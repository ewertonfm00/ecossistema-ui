> **Status:** Draft

## Story

**Como** desenvolvedor implementando formulários complexos nos módulos do ecossistema (Estética IA, Gestão, Auth),
**quero** controles de formulário avançados e utilitários visuais prontos no `@aiox/ui`,
**para que** eu possa compor telas com checkboxes, radios, switches, textareas, selects, tooltips e skeletons sem recriar esses padrões em cada módulo.

## Contexto

Story 0.3 entregou as 4 moléculas core. Esta story entrega os 8 **átomos de prioridade média** — controles de formulário mais complexos e utilitários visuais que completam o vocabulário de formulários e UX do design system:

- **Checkbox** — seleção múltipla com estado indeterminate
- **Radio** — seleção única em grupo via `name`
- **Switch** — toggle on/off com transição suave
- **Textarea** — campo de texto multilinha com redimensionamento controlado
- **Select** — dropdown com options tipadas
- **Tooltip** — dica contextual via CSS puro (sem popper.js)
- **Skeleton** — placeholder animado de carregamento
- **Divider** — separador horizontal/vertical com label opcional

**Regra de composição:** todos os átomos usam APENAS tokens do `tailwind-preset.js`. Zero valores de cor, espaçamento ou tipografia hardcoded.

**Stack:** React 18 + TypeScript strict + TailwindCSS v3 + Vitest + cva (mesmo padrão das Stories 0.2 e 0.3)

### 3. `Checkbox` — seleção múltipla

**Props:**
```ts
interface CheckboxProps {
  id: string
  checked?: boolean
  indeterminate?: boolean   // estado visual de seleção parcial
  disabled?: boolean
  label?: string            // texto ao lado do checkbox (via htmlFor interno)
  onChange?: (checked: boolean) => void
  className?: string
}
```

**Comportamento:**
- Elemento base: `<input type="checkbox">` nativo — acessibilidade nativa garantida
- `indeterminate`: aplicado via `ref.current.indeterminate = true` em `useEffect` — não existe como atributo HTML direto
- `disabled`: opacidade reduzida (`opacity-50 cursor-not-allowed`) + bloqueia `onChange`
- Label: quando `label` fornecido, renderizar `<label htmlFor={id}` ao lado do input
- Estilo do input: `accent-primary-700` via classe CSS customizada — sem modificar cor nativa do OS além do `accent-color`
- Wrapper: `<div className="flex items-center gap-2">`

**Tokens a usar:**
- Cor do checkmark: `accent-primary-700` (definir via CSS: `accent-color: theme('colors.primary.700')`)
- Foco: `focus:ring-2 focus:ring-primary-700 focus:ring-offset-2`
- Borda: `border-neutral-300` no estado padrão

### 4. `Radio` — seleção única em grupo

**Props:**
```ts
interface RadioProps {
  id: string
  name: string             // agrupa radios do mesmo grupo
  value: string
  checked?: boolean
  disabled?: boolean
  label?: string
  onChange?: (value: string) => void
  className?: string
}
```

**Comportamento:**
- Elemento base: `<input type="radio">` nativo
- Agrupamento: via atributo `name` nativo do HTML — sem contexto React necessário
- `onChange`: chama com `value` (não com event) para API consistente com os outros átomos
- `disabled`: `opacity-50 cursor-not-allowed`
- Label: mesmo padrão do Checkbox — `<label htmlFor={id}>` quando `label` fornecido
- Estilo: `accent-primary-700` (mesmo mecanismo do Checkbox)
- Foco: `focus:ring-2 focus:ring-primary-700 focus:ring-offset-2`

### 5. `Switch` — toggle on/off

**Props:**
```ts
interface SwitchProps {
  checked?: boolean
  disabled?: boolean
  label?: string            // label ao lado do switch
  labelPosition?: 'left' | 'right'   // padrão: 'right'
  onChange?: (checked: boolean) => void
  className?: string
}
```

**Comportamento:**
- Implementação: `<button role="switch" aria-checked={checked}>` — SEM `<input type="checkbox">` para ter controle visual total
- Estado OFF: `bg-neutral-200`
- Estado ON: `bg-primary-700`
- Transição: `transition-colors duration-200` entre os dois estados
- Thumb (círculo branco): `bg-white rounded-full shadow-sm` — translada via `translate-x` (OFF: `translate-x-0`, ON: `translate-x-5`)
- Thumb transition: `transition-transform duration-200`
- Dimensões do track: `w-10 h-6 rounded-full` (40×24px)
- Thumb: `w-5 h-5` (20×20px), com margin de 2px dentro do track
- `disabled`: `opacity-50 cursor-not-allowed` — não chama `onChange`
- Label: renderizado fora do button, em `<span>` com `text-sm text-neutral-700`

**Acessibilidade:**
- `role="switch"` no `<button>`
- `aria-checked={checked}` dinâmico
- `aria-label` quando `label` não fornecido: obrigatório via prop ou erro TypeScript

### 6. `Textarea` — campo de texto multilinha

**Props:**
```ts
interface TextareaProps extends React.TextareaHTMLAttributes<HTMLTextAreaElement> {
  state?: 'default' | 'error' | 'disabled'
  resize?: 'none' | 'vertical' | 'both'   // padrão: 'vertical'
  rows?: number                             // padrão: 3
  className?: string
}
```

**Comportamento:**
- Wrapper: `<textarea>` nativo com `extends React.TextareaHTMLAttributes` — herda todas as props nativas (placeholder, maxLength, etc.)
- `state='disabled'`: aplica `disabled` nativo + classes visuais (`bg-neutral-50 text-neutral-400 cursor-not-allowed`)
- `state='error'`: `border-error-500 focus:ring-error-500`
- `state='default'`: `border-neutral-300 focus:ring-primary-700`
- Resize: `resize-none` | `resize-y` | `resize` via classe Tailwind
- Estilos base: `w-full rounded-md border px-3 py-2 text-sm text-neutral-900 focus:outline-none focus:ring-2 focus:ring-offset-0`
- Background padrão: `bg-white`

**Tokens a usar:**
- Borda erro: `border-error-500`
- Ring erro: `ring-error-500`
- Ring padrão: `ring-primary-700`
- Texto desabilitado: `text-neutral-400`
- Background desabilitado: `bg-neutral-50`

### 7. `Select` — dropdown nativo

**Props:**
```ts
interface SelectOption {
  value: string
  label: string
  disabled?: boolean
}

interface SelectProps {
  options: SelectOption[]
  value?: string
  placeholder?: string     // exibido como <option value="" disabled> quando fornecido
  state?: 'default' | 'error' | 'disabled'
  onChange?: (value: string) => void
  className?: string
}
```

**Comportamento:**
- Elemento base: `<select>` nativo — sem dropdown customizado
- `placeholder`: renderizado como `<option value="" disabled selected hidden>{placeholder}</option>`
- `state='disabled'`: atributo `disabled` nativo + `bg-neutral-50 text-neutral-400 cursor-not-allowed`
- `state='error'`: `border-error-500 focus:ring-error-500`
- `onChange`: chama com `event.target.value` (string) — não com o event completo
- Ícone de chevron: via CSS (`appearance-none` + `bg-[url(...)]`) ou ícone SVG absolutamente posicionado
- Estilos base: `w-full rounded-md border px-3 py-2 text-sm text-neutral-900 focus:outline-none focus:ring-2 bg-white appearance-none`
- Wrapper relativo: `<div className="relative">` para posicionar ícone chevron

**Tokens a usar:**
- Mesmos tokens de borda/ring que o Textarea

### 8. `Tooltip` — dica contextual via CSS

**Props:**
```ts
interface TooltipProps {
  content: string           // texto da tooltip (sem JSX para manter simples)
  position?: 'top' | 'bottom' | 'left' | 'right'   // padrão: 'top'
  children: React.ReactElement   // o trigger — qualquer elemento React
  className?: string
}
```

**Comportamento:**
- Implementação: CSS puro — `group` no wrapper + `group-hover:visible` / `invisible` na tooltip
- Sem JavaScript para posicionamento — CSS `absolute` relativo ao wrapper `relative`
- Wrapper: `<div className="relative inline-flex">` com `group` do Tailwind
- Tooltip bubble: `<span>` absolutamente posicionado, `z-50`, `invisible group-hover:visible`
- Posicionamento por `position`:
  - `top`: `bottom-full left-1/2 -translate-x-1/2 mb-2`
  - `bottom`: `top-full left-1/2 -translate-x-1/2 mt-2`
  - `left`: `right-full top-1/2 -translate-y-1/2 mr-2`
  - `right`: `left-full top-1/2 -translate-y-1/2 ml-2`
- Estilo da bubble: `bg-neutral-900 text-white text-xs rounded px-2 py-1 whitespace-nowrap`
- Arrow (triângulo CSS): `::before` via classe customizada ou `<span>` extra com borda colorida

**Acessibilidade:**
- `role="tooltip"` na bubble
- `aria-describedby` no children apontando para o id da bubble
- `id` gerado via `useId()` do React 18

**Limitação aceita:** tooltip via CSS puro não funciona em teclado (`:focus-within` pode ajudar mas não é escopo desta story). Documentar como limitação.

### 9. `Skeleton` — placeholder de carregamento

**Props:**
```ts
interface SkeletonProps {
  width?: string | number    // padrão: '100%'
  height?: string | number   // padrão: '1rem'
  variant?: 'rect' | 'circle' | 'text'   // padrão: 'rect'
  className?: string
}
```

**Comportamento:**
- Elemento: `<div>` com `animate-pulse bg-neutral-200`
- `variant='rect'`: `rounded-md` (canto arredondado padrão)
- `variant='circle'`: `rounded-full` — width e height iguais formam círculo
- `variant='text'`: `rounded` com `height` padrão de `0.875rem` (text-sm) e `width` em `%`
- `width` e `height` aplicados via style inline quando fornecidos como número (converte para `px`) ou string (`%`, `rem`, `px`)
- Sem children — é um elemento vazio de placeholder

**Acessibilidade:**
- `aria-hidden="true"` — skeleton não tem significado semântico
- `role="presentation"` opcional mas recomendado

### 10. `Divider` — separador

**Props:**
```ts
interface DividerProps {
  orientation?: 'horizontal' | 'vertical'   // padrão: 'horizontal'
  label?: string    // texto central opcional (só para horizontal)
  className?: string
}
```

**Comportamento:**
- Horizontal sem label: `<hr className="border-neutral-200 my-4">`
- Horizontal com label: `<div className="flex items-center gap-3"><hr className="flex-1"/><span className="text-xs text-neutral-400">{label}</span><hr className="flex-1"/></div>`
- Vertical: `<div className="inline-block h-full w-px bg-neutral-200 mx-2">` — height controlado pelo container pai
- Sem `<hr>` para vertical — `<div>` com `role="separator"` e `aria-orientation="vertical"`

**Acessibilidade:**
- `role="separator"` em todos os casos
- `aria-orientation` dinâmico conforme prop

## Notas técnicas

### Padrão de `accent-color` para Checkbox e Radio

```tsx
// Abordagem recomendada — CSS customizado via Tailwind plugin ou inline style:
<input
  type="checkbox"
  style={{ accentColor: 'var(--color-primary-700)' }}
  className="w-4 h-4 rounded border-neutral-300 focus:ring-2 focus:ring-primary-700"
/>
```

Se `tailwind-preset.js` não expuser `--color-primary-700` como variável CSS, usar o valor direto do token. Verificar antes de implementar.

### `indeterminate` no Checkbox

O estado `indeterminate` não existe como atributo HTML. Deve ser aplicado via ref:

```tsx
const inputRef = useRef<HTMLInputElement>(null)

useEffect(() => {
  if (inputRef.current) {
    inputRef.current.indeterminate = indeterminate ?? false
  }
}, [indeterminate])
```

### Switch sem `<input type="checkbox">`

O Switch usa `<button role="switch">` para ter controle visual completo. Isso requer que o consumidor gerencie o estado (controlled component pattern — mesmo padrão dos outros átomos).

### Tooltip e `useId`

`useId()` do React 18 gera IDs únicos para evitar colisão quando múltiplos Tooltips existem na mesma página. Não usar `Math.random()` ou contadores manuais.

### Select e ícone chevron

Usar `appearance-none` no select nativo e adicionar ícone `<Icon name="ChevronDown">` absolutamente posicionado no wrapper. Isso garante visual consistente sem dropdown customizado.

### Atualização do `tailwind-preset.js`

Se `shadow-elevation-sm` ainda não existe, adicionar nesta story (identificado em Story 0.3 QA LOW-2):

```js
boxShadow: {
  'elevation-sm': '0 1px 3px 0 rgb(0 0 0 / 0.1), 0 1px 2px -1px rgb(0 0 0 / 0.1)',
  // ...demais tokens
}
```
