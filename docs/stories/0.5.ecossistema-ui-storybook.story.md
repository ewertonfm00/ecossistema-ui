## Story

**Como** desenvolvedor integrando `@aiox/ui` em qualquer módulo do ecossistema,
**quero** um Storybook com stories para todos os componentes,
**para que** eu possa explorar variantes, props e comportamentos sem precisar ler o código-fonte de cada componente.

## Contexto

Após Stories 0.2–0.4, `@aiox/ui` terá 15+ componentes. Sem documentação interativa, novos devs dependem de leitura de código + TypeScript para descobrir variantes e comportamentos. Esta story configura o **Storybook** com:

- Configuração do `@storybook/react-vite` integrada ao Tailwind via PostCSS
- Stories `.stories.tsx` para todos os componentes existentes (15+ componentes)
- Mínimo 2 stories por componente: **Default** + variantes principais
- Script `storybook` no `package.json`
- **Chromatic** para visual regression (opcional — aguardar aprovação)

**Stack:** `@storybook/react-vite` + `@storybook/addon-essentials` · Tailwind via PostCSS · TypeScript strict

### 3. `.storybook/main.ts` — configuração

```ts
import type { StorybookConfig } from '@storybook/react-vite'

const config: StorybookConfig = {
  stories: ['../src/**/*.stories.@(ts|tsx)'],
  addons: ['@storybook/addon-essentials'],
  framework: {
    name: '@storybook/react-vite',
    options: {},
  },
}

export default config
```

### 4. `.storybook/preview.ts` — Tailwind + decorators globais

```ts
import type { Preview } from '@storybook/react'
import '../src/index.css'    // ou o entry point do Tailwind do projeto

const preview: Preview = {
  parameters: {
    actions: { argTypesRegex: '^on[A-Z].*' },
    controls: {
      matchers: {
        color: /(background|color)$/i,
        date: /Date$/i,
      },
    },
    backgrounds: {
      default: 'light',
      values: [
        { name: 'light', value: '#f9fafb' },   // neutral-50 do preset
        { name: 'white', value: '#ffffff' },
        { name: 'dark', value: '#111827' },
      ],
    },
  },
}

export default preview
```

**Nota:** O entry point do Tailwind deve importar o `tailwind-preset.js` do projeto. Verificar se existe `src/index.css` ou `src/styles/globals.css` antes de implementar.

### 7. Padrão de arquivo `.stories.tsx`

```tsx
import type { Meta, StoryObj } from '@storybook/react'
import { Button } from './Button'

const meta: Meta<typeof Button> = {
  title: 'Atoms/Button',
  component: Button,
  tags: ['autodocs'],
  argTypes: {
    variant: {
      control: 'select',
      options: ['primary', 'secondary', 'outline', 'ghost', 'danger'],
    },
  },
}

export default meta
type Story = StoryObj<typeof Button>

export const Default: Story = {
  args: {
    children: 'Clique aqui',
    variant: 'primary',
    size: 'md',
  },
}

export const AllVariants: Story = {
  render: () => (
    <div className="flex gap-2 flex-wrap">
      {/* ...variantes */}
    </div>
  ),
}
```

**Convenções de `title`:**
- Átomos: `Atoms/ComponentName`
- Moléculas: `Molecules/ComponentName`
- Organismos: `Organisms/ComponentName`

**`tags: ['autodocs']`** — obrigatório em todos para gerar documentação automática.

## Notas técnicas

### Tailwind no Storybook

O Storybook com `@storybook/react-vite` usa o Vite do projeto automaticamente. O `vite.config.ts` existente já deve ter o plugin do Tailwind (`@vitejs/plugin-react` + `tailwindcss`). Verificar se `tailwindcss` já está no `content` do `tailwind.config.js` incluindo `./src/**/*.{stories.tsx}` se necessário:

```js
// tailwind.config.js
module.exports = {
  content: [
    './src/**/*.{ts,tsx}',
    './src/**/*.stories.tsx',   // garantir que stories são escaneadas
  ],
  presets: [require('./tailwind-preset.js')],
}
```

### Dados mock para DataTable

Criar dados mock diretamente no arquivo `.stories.tsx` — sem arquivo separado de fixtures:

```tsx
const mockUsers = Array.from({ length: 35 }, (_, i) => ({
  id: i + 1,
  name: `Usuário ${i + 1}`,
  email: `user${i + 1}@example.com`,
  status: i % 3 === 0 ? 'inactive' : 'active',
}))
```

### AppShell stories

AppShell ocupa `h-screen`. No Storybook, definir `parameters.layout = 'fullscreen'` na story do AppShell:

```tsx
export const Default: Story = {
  parameters: { layout: 'fullscreen' },
  // ...
}
```

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| Storybook 8.x incompatível com Vite config existente | Médio | Verificar `@storybook/react-vite` peer deps antes de instalar; ajustar `vite.config.ts` se necessário |
| Tailwind não processando classes das stories | Médio | Adicionar `./src/**/*.stories.tsx` ao `content` do tailwind.config.js |
| AppShell com `h-screen` quebrando layout no Storybook iframe | Alto | Usar `parameters.layout = 'fullscreen'` na story — documentado nesta story |
| Chromatic token ausente se aprovado | Baixo | Adicionar `CHROMATIC_PROJECT_TOKEN` ao `.env.local` e GitHub Secrets separadamente |
| Stories importando tipos não exportados do `src/index.ts` | Baixo | Garantir que todos os tipos interfaces (DataTableColumn etc.) estão exportados no barrel |
