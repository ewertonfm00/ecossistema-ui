## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema,
**quero** componentes `Modal` e `Drawer` acessíveis e compostos com o design system `@aiox/ui`,
**para que** eu possa exibir conteúdo em sobreposição e painéis laterais sem reimplementar trap de foco, Escape para fechar e acessibilidade em cada módulo.

---

## Contexto

Formulários de confirmação, detalhes de registro e painéis de configuração são padrões recorrentes em todos os módulos do ecossistema (Estética IA, ML Lab, Auth). Atualmente cada módulo teria que reimplementar overlay + trap de foco + acessibilidade.

`Modal` e `Drawer` são organismos porque orquestram múltiplos componentes (overlay, Card, Button, Icon) e gerenciam estado de foco. Ambos usam `ReactDOM.createPortal` para renderizar no `document.body`, garantindo z-index correto independente de onde são montados na árvore.

**Decisão arquitetural:** sem dependências externas de acessibilidade (Radix, Headless UI) — implementação própria com `focus-trap` nativo via `tabIndex` e listeners de teclado. Sem `framer-motion` — animações via classes CSS do Tailwind (`transition-opacity`, `transition-transform`).

### 2. `Modal` — diálogo acessível com overlay

**Interface:**
```ts
interface ModalProps {
  open: boolean
  onClose: () => void
  title: string
  size?: 'sm' | 'md' | 'lg'   // padrão: 'md'
  hideCloseButton?: boolean    // padrão: false
  children: React.ReactNode
  footer?: React.ReactNode     // slot para botões de ação
}
```

**Tamanhos (`size`):**
| Valor | `max-width` equivalente |
|-------|------------------------|
| `sm`  | `max-w-sm` (384px)     |
| `md`  | `max-w-lg` (512px)     |
| `lg`  | `max-w-2xl` (672px)    |

**Comportamento:**
- Renderiza via `ReactDOM.createPortal` no `document.body`
- Quando `open=false`: não monta no DOM (condicional — não apenas `display:none`)
- Overlay: `fixed inset-0 bg-black/50` — clique no overlay chama `onClose`
- Clique dentro do painel NÃO fecha (stopPropagation no container)
- Pressionar `Escape` chama `onClose`
- Ao abrir: foco vai para o primeiro elemento focável dentro do modal (ou para o painel se não houver)
- Ao fechar: foco retorna ao elemento que estava ativo antes da abertura
- `Body scroll lock`: enquanto modal aberto, adicionar `overflow-hidden` ao `document.body`; remover ao fechar

**Trap de foco (obrigatório):**
- `Tab` cicla apenas entre elementos focáveis DENTRO do modal
- `Shift+Tab` cicla ao contrário
- Elementos focáveis: `a[href]`, `button:not([disabled])`, `input:not([disabled])`, `select:not([disabled])`, `textarea:not([disabled])`, `[tabindex]:not([tabindex="-1"])`

**Estrutura JSX:**
```tsx
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title-{id}"
  className="fixed inset-0 z-50 flex items-center justify-center p-4"
>
  {/* Overlay */}
  <div className="absolute inset-0 bg-black/50" onClick={onClose} aria-hidden="true" />
  {/* Painel */}
  <div className={cn('relative bg-white rounded-lg shadow-xl w-full', sizeClass)}>
    {/* Header */}
    <div className="flex items-center justify-between p-6 border-b border-neutral-200">
      <h2 id="modal-title-{id}" className="text-lg font-semibold text-neutral-900">{title}</h2>
      {!hideCloseButton && (
        <button onClick={onClose} aria-label="Fechar modal" className="...">
          <Icon name="X" size={20} />
        </button>
      )}
    </div>
    {/* Body */}
    <div className="p-6">{children}</div>
    {/* Footer */}
    {footer && (
      <div className="flex justify-end gap-3 p-6 border-t border-neutral-200">{footer}</div>
    )}
  </div>
</div>
```

**Animação (CSS only):**
- Overlay: `transition-opacity duration-200`
- Painel: `transition-all duration-200 scale-95 → scale-100` ao abrir

**Stories (Storybook):**
- `Default` — modal md aberto com texto e footer com "Cancelar" + "Confirmar"
- `Small` — size sm com mensagem de confirmação simples
- `Large` — size lg com formulário de múltiplos campos
- `NoCloseButton` — `hideCloseButton={true}` (usuário só pode fechar via footer)

**Testes (mínimo 2 por componente):**
- [ ] Smoke: renderiza quando `open=true`
- [ ] Não renderiza quando `open=false`
- [ ] Título está presente com `role="dialog"` e `aria-labelledby` correto
- [ ] Pressionar Escape chama `onClose`
- [ ] Clicar no overlay chama `onClose`
- [ ] Clicar dentro do painel NÃO chama `onClose`

### 3. `Drawer` — painel lateral deslizante

**Interface:**
```ts
interface DrawerProps {
  open: boolean
  onClose: () => void
  title: string
  side?: 'right' | 'left'   // padrão: 'right'
  width?: 'sm' | 'md' | 'lg' // padrão: 'md'
  hideCloseButton?: boolean   // padrão: false
  children: React.ReactNode
  footer?: React.ReactNode
}
```

**Larguras (`width`):**
| Valor | `width` equivalente |
|-------|---------------------|
| `sm`  | `w-80` (320px)      |
| `md`  | `w-96` (384px)      |
| `lg`  | `w-[480px]`         |

**Comportamento:**
- Renderiza via `ReactDOM.createPortal` no `document.body`
- Quando `open=false`: não monta no DOM
- Overlay: `fixed inset-0 bg-black/50` — clique no overlay chama `onClose`
- Pressionar `Escape` chama `onClose`
- Trap de foco idêntico ao Modal
- Foco vai ao painel ao abrir; retorna ao trigger ao fechar
- Body scroll lock enquanto aberto

**Animação (CSS only, baseada em `side`):**
- `right`: `translate-x-full → translate-x-0` (desliza da direita)
- `left`: `-translate-x-full → translate-x-0` (desliza da esquerda)
- `transition-transform duration-300 ease-in-out`

**Estrutura JSX:**
```tsx

  role="dialog"
  aria-modal="true"
  aria-labelledby="drawer-title-{id}"
  className={cn('fixed inset-y-0 z-50 flex flex-col bg-white shadow-xl', sideClass, widthClass)}
>
  {/* Header */}
  <div className="flex items-center justify-between p-6 border-b border-neutral-200">
    <h2 id="drawer-title-{id}" className="text-lg font-semibold text-neutral-900">{title}</h2>
    {!hideCloseButton && (
      <button onClick={onClose} aria-label="Fechar painel">
        <Icon name="X" size={20} />
      </button>
    )}
  </div>
  {/* Body */}
  <div className="flex-1 overflow-y-auto p-6">{children}</div>
  {/* Footer */}
  {footer && (
    <div className="flex justify-end gap-3 p-6 border-t border-neutral-200">{footer}</div>
  )}
</div>
```

**Stories (Storybook):**
- `Default` — drawer direito md com conteúdo de texto
- `Left` — `side="left"` deslizando da esquerda
- `Large` — `width="lg"` com formulário completo
- `WithFooter` — footer com "Cancelar" + "Salvar"

## Notas técnicas

### Portal obrigatório

Ambos usam `ReactDOM.createPortal(content, document.body)`. Isso garante que o z-index funcione corretamente mesmo quando o componente está dentro de um container com `overflow:hidden` ou `position:relative`. Sem portal, o overlay não cobre o AppShell corretamente.

### ID único para `aria-labelledby`

Usar `React.useId()` (React 18) para gerar ids únicos: `const id = useId()`. Isso evita colisão quando múltiplos modais são montados no SSR ou em testes.

### Trap de foco — implementação sem biblioteca

```ts
const FOCUSABLE = 'a[href],button:not([disabled]),input:not([disabled]),select:not([disabled]),textarea:not([disabled]),[tabindex]:not([tabindex="-1"])'

function trapFocus(container: HTMLElement, e: KeyboardEvent) {
  const focusable = Array.from(container.querySelectorAll<HTMLElement>(FOCUSABLE))
  const first = focusable[0]
  const last = focusable[focusable.length - 1]
  if (e.key === 'Tab') {
    if (e.shiftKey && document.activeElement === first) {
      e.preventDefault(); last.focus()
    } else if (!e.shiftKey && document.activeElement === last) {
      e.preventDefault(); first.focus()
    }
  }
}
```

### Body scroll lock

```ts
useEffect(() => {
  if (open) {
    document.body.style.overflow = 'hidden'
    return () => { document.body.style.overflow = '' }
  }
}, [open])
```

### Não usar `display:none` para fechar

Quando `open=false`, o componente retorna `null` (não monta). Isso garante que listeners de teclado e efeitos sejam limpos pelo React.
