## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema,
**quero** um `DataTable` com sorting por coluna, filtros por coluna, seleção de linhas com checkbox, export CSV e empty state com CTA,
**para que** eu possa construir listas de produção (clientes, agendamentos, transações, datasets) sem reimplementar lógica de tabela em cada módulo.

---

## Contexto

O `DataTable` atual (Story 0.4) tem paginação e loading skeleton — suficiente para protótipos, mas bloqueante para uso real. Estética IA precisa ordenar clientes por nome/data, filtrar agendamentos por status, e selecionar múltiplos para ações em lote. AIOX Gestão precisa exportar relatórios financeiros. ML Lab precisa selecionar linhas de dataset para operações.

Sem essas features no design system, cada módulo implementa a própria versão — divergência visual garantida.

**Decisão arquitetural:** Toda a lógica de sorting, filtering e seleção é client-side sobre o array `data` completo. Server-side pagination/sorting é responsabilidade dos módulos consumidores — o componente gerencia apenas o estado local. Limite documentado: ~500 linhas sem virtualização.

### 3. Sorting por coluna

#### 3a. Estado interno

```ts
type SortDirection = 'asc' | 'desc' | null
const [sortKey, setSortKey] = useState<string | null>(null)
const [sortDir, setSortDir] = useState<SortDirection>(null)
```

#### 3b. Ciclo de clique no header

| Clique | Estado anterior | Estado novo |
|--------|----------------|-------------|
| 1º | null | ASC |
| 2º | ASC | DESC |
| 3º | DESC | null (ordem original) |
| Clique em outra coluna | qualquer | ASC na nova coluna |

#### 3c. Ícones no header (usar `Icon` do design system)

| Estado | Ícone lucide |
|--------|-------------|
| Neutro (sortable, não ativo) | `ArrowUpDown` |
| ASC ativo | `ArrowUp` |
| DESC ativo | `ArrowDown` |

```tsx
{col.sortable && (
  <button
    onClick={() => handleSort(String(col.key))}
    className="inline-flex items-center gap-1 hover:text-neutral-900"
    aria-label={`Ordenar por ${col.header}`}
  >
    {col.header}
    <Icon
      name={sortKey === String(col.key) ? (sortDir === 'asc' ? 'ArrowUp' : 'ArrowDown') : 'ArrowUpDown'}
      size={14}
      className={sortKey === String(col.key) ? 'text-primary-600' : 'text-neutral-400'}
    />
  </button>
)}
```

#### 3d. Lógica de sort

```ts
function sortData(rows: T[]): T[] {
  if (!sortKey || !sortDir) return rows
  return [...rows].sort((a, b) => {
    const aVal = (a as Record<string, unknown>)[sortKey]
    const bVal = (b as Record<string, unknown>)[sortKey]
    if (aVal == null) return 1
    if (bVal == null) return -1
    const cmp = String(aVal).localeCompare(String(bVal), 'pt-BR', { numeric: true, sensitivity: 'base' })
    return sortDir === 'asc' ? cmp : -cmp
  })
}
```

- Sorting opera sobre o `data` **completo** antes de filtrar e paginar
- `localeCompare` com `pt-BR` + `numeric: true` ordena "Item 10" após "Item 9"
- Valores `null`/`undefined` vão para o final independente da direção

Acima da tabela, quando `activeFilterCount > 0`:

```tsx

  <div className="flex items-center gap-2 mb-2">
    <Badge variant="secondary">{activeFilterCount} {activeFilterCount === 1 ? 'filtro ativo' : 'filtros ativos'}</Badge>

      onClick={() => setFilters({})}
      className="text-xs text-primary-600 hover:underline"
    >
      Limpar filtros
    </button>
  </div>

```

#### 4e. Lógica de filtering

```ts
function filterData(rows: T[]): T[] {
  return rows.filter(row =>
    Object.entries(filters).every(([key, val]) => {
      if (!val) return true
      const cellVal = String((row as Record<string, unknown>)[key] ?? '')
      return cellVal.toLowerCase().includes(val.toLowerCase())
    })
  )
}
```

**Pipeline de dados:** `data` → `filterData` → `sortData` → `paginação` → `renderização`

### 6. Export CSV

#### 6a. Botão

Renderizado no canto superior direito do componente quando `exportable=true`:

```tsx
<button
  onClick={handleExport}
  className="flex items-center gap-1 px-3 py-1.5 text-sm border border-neutral-200 rounded-md hover:bg-neutral-50 text-neutral-700"
>
  <Icon name="Download" size={14} />
  Exportar CSV
</button>
```

#### 6b. Lógica de export

```ts
function handleExport() {
  // Dados a exportar: selecionados > filtrados > todos
  const rows = selected.size > 0
    ? Array.from(selected).map(i => data[i]).filter((r): r is T => r !== undefined)
    : filterData(data)

  const headers = columns
    .filter(col => col.key !== '__actions__')
    .map(col => col.header)

  const csvRows = rows.map(row =>
    columns
      .filter(col => col.key !== '__actions__')
      .map(col => {
        const val = (row as Record<string, unknown>)[String(col.key)]
        const str = String(val ?? '')
        // Escapar aspas duplas e envolver em aspas se necessário
        return str.includes(',') || str.includes('"') || str.includes('\n')
          ? `"${str.replace(/"/g, '""')}"`
          : str
      })
      .join(',')
  )

  // UTF-8 BOM para Excel PT-BR reconhecer acentos
  const bom = '﻿'
  const csv = bom + [headers.join(','), ...csvRows].join('\r\n')
  const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' })
  const url = URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = `${exportFilename ?? 'export'}.csv`
  link.click()
  URL.revokeObjectURL(url)
}
```

## Notas técnicas

### Pipeline de dados (ordem obrigatória)

```
data (prop) → filterData() → sortData() → sliceForPage() → render
```

Filtering antes de sorting garante que o sort opere apenas nos dados visíveis. Paginação no final garante que "página 1 de N" reflita o filtro aplicado. Resetar para página 1 sempre que `filters` ou `sortKey` mudar.

### Seleção por índice original (não por página)

`selected` armazena índices do array `data` **original** (antes de filter/sort/paginate). Isso garante que seleções feitas na página 1 persistam ao navegar para página 2 e voltar. A conversão para linha real é `data[i]`.

### Export CSV — `URL.createObjectURL` em testes

Em ambiente Vitest/JSDOM, `URL.createObjectURL` não existe. Mockar antes dos testes:

```ts
beforeAll(() => {
  global.URL.createObjectURL = vi.fn(() => 'blob:mock')
  global.URL.revokeObjectURL = vi.fn()
})
```

### `indeterminate` via ref

O estado `indeterminate` do checkbox não é uma prop React — requer acesso direto ao DOM via `ref`:

```tsx
const headerCheckRef = useRef<HTMLInputElement>(null)
useEffect(() => {
  if (headerCheckRef.current) {
    headerCheckRef.current.indeterminate = partialSelected
  }
}, [partialSelected])
```

### `localeCompare` para PT-BR

```ts
String(aVal).localeCompare(String(bVal), 'pt-BR', { numeric: true, sensitivity: 'base' })
```

- `numeric: true` — "Item 10" ordena após "Item 9" (não "Item 1" → "Item 10" → "Item 9")
- `sensitivity: 'base'` — ignora maiúsculas/minúsculas e acentos na ordenação

### Filtering sobre valor bruto (não render)

O `render` pode retornar JSX — comparar `String(JSX)` sempre retorna `[object Object]`. Filtrar sempre sobre o valor bruto: `(row as Record<string, unknown>)[key]`.
