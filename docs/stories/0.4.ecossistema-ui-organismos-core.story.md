## Story

**Como** desenvolvedor implementando qualquer módulo do ecossistema (Estética IA, Auth, Gestão, ML Lab),
**quero** organismos de layout prontos no `@aiox/ui` para compor a estrutura visual das telas,
**para que** eu possa montar AppShell + SidebarNav + PageHeader + DataTable sem recriar a arquitetura de layout em cada módulo.

## Contexto

Stories 0.2, 0.3 e 0.3b entregaram os átomos e moléculas. Esta story entrega os 4 **organismos core** — componentes de alta composição que estruturam layouts completos de aplicação:

- **AppShell** — layout root com sidebar + main area
- **SidebarNav** — navegação lateral colapsável com ícones
- **PageHeader** — cabeçalho de página com breadcrumb e actions
- **DataTable** — tabela de dados com paginação simples

**Regra de composição:** organismos usam APENAS átomos e moléculas das Stories 0.2, 0.3 e 0.3b. Nenhuma dependência de UI externa. HTML semântico obrigatório (`<aside>`, `<main>`, `<header>`, `<nav>`, `<table>`).

**Stack:** React 18 + TypeScript strict + TailwindCSS v3 + Vitest + cva (mesmo padrão das stories anteriores)

### 3. `AppShell` — layout root

**Props:**
```ts
interface AppShellProps {
  sidebar: React.ReactNode    // slot para SidebarNav ou conteúdo lateral
  children: React.ReactNode   // conteúdo da área principal (main)
  className?: string
}
```

**Estrutura HTML obrigatória:**
```html
<div class="flex h-screen overflow-hidden bg-neutral-50">
  <aside class="flex-shrink-0">
    <!-- sidebar slot -->
  </aside>
  <main class="flex-1 overflow-y-auto">
    <!-- children slot -->
  </main>
</div>
```

**Comportamento:**
- `<aside>`: `flex-shrink-0` — não encolhe com o conteúdo do `<main>`
- `<main>`: `flex-1 overflow-y-auto` — ocupa o restante e rola independentemente
- `<div>` root: `flex h-screen overflow-hidden` — viewport completo sem scroll externo
- Responsividade: não é escopo desta story — apenas layout desktop (≥1024px)
- AppShell não conhece SidebarNav — recebe qualquer `React.ReactNode` no slot `sidebar`

**Acessibilidade:**
- `<aside>` com `aria-label="Navegação lateral"`
- `<main>` com `id="main-content"` para skip navigation

### 4. `SidebarNav` — navegação lateral

**Props:**
```ts
interface SidebarNavItem {
  label: string
  icon: string          // nome do ícone Lucide (string passada para <Icon name>)
  href: string
  active?: boolean
  disabled?: boolean
}

interface SidebarNavProps {
  items: SidebarNavItem[]
  logo?: React.ReactNode   // slot para logo ou marca
  collapsed?: boolean      // padrão: false
  onToggleCollapse?: () => void   // callback para alternar collapsed
  className?: string
}
```

**Comportamento:**
- Largura expanded: `w-60` (240px)
- Largura collapsed: `w-16` (64px)
- Transição de largura: `transition-all duration-300`
- `collapsed=true`: exibe apenas ícones — texto (`label`) fica invisível com `sr-only` ou `hidden`
- `collapsed=false`: exibe ícone + label em linha
- Item ativo (`active=true`): `bg-primary-50 text-primary-700 font-medium`
- Item padrão: `text-neutral-600 hover:bg-neutral-100 hover:text-neutral-900`
- Item desabilitado: `opacity-50 cursor-not-allowed pointer-events-none`
- Ícone: usa `<Icon name={item.icon}>` do átomo existente
- Botão de toggle: `<button>` com `<Icon name="ChevronLeft">` / `<Icon name="ChevronRight">` conforme estado
- Logo slot: renderizado no topo da sidebar, antes dos items
- Wrapper da sidebar: `<nav>` semântico

**Estrutura HTML:**
```html
<nav class="flex flex-col h-full bg-white border-r border-neutral-200">
  <div class="logo-slot">...</div>
  <ul class="flex-1 px-2 py-4 space-y-1">
    <li><a href="...">...</a></li>
  </ul>
  <div class="toggle-button">...</div>
</nav>
```

**Acessibilidade:**
- `<nav>` com `aria-label="Menu principal"`
- `<ul>` + `<li>` para lista de itens
- `<a>` para links navegáveis (não `<button>`)
- Item ativo: `aria-current="page"` no `<a>`
- Botão toggle: `aria-expanded={!collapsed}` + `aria-label="Recolher menu"` / `"Expandir menu"`

### 5. `PageHeader` — cabeçalho de página

**Props:**
```ts
interface BreadcrumbItem {
  label: string
  href?: string    // se presente, renderiza como link; senão, como span
}

interface PageHeaderProps {
  title: string
  breadcrumbs?: BreadcrumbItem[]
  actions?: React.ReactNode   // slot para botões de ação (ex: <Button>Novo</Button>)
  className?: string
}
```

**Estrutura HTML:**
```html
<header class="flex items-start justify-between px-6 py-4 border-b border-neutral-200 bg-white">
  <div>
    <!-- breadcrumb (opcional) -->
    <nav aria-label="Breadcrumb">
      <ol>
        <li>...</li>
        <li aria-current="page">Título atual</li>
      </ol>
    </nav>
    <!-- título -->
    <h1 class="text-2xl font-semibold text-neutral-900">...</h1>
  </div>
  <div class="actions-slot">
    <!-- React.ReactNode -->
  </div>
</header>
```

**Comportamento:**
- Breadcrumb: separador entre itens via `<span aria-hidden="true"> / </span>` ou ícone `ChevronRight`
- Último breadcrumb item: `aria-current="page"`, renderizado como `<span>` (não link)
- `actions`: slot `React.ReactNode` — PageHeader não conhece os botões
- Background: `bg-white` com `border-b border-neutral-200`
- Padding: `px-6 py-4`
- Título: `text-2xl font-semibold text-neutral-900`
- Breadcrumb texto: `text-sm text-neutral-500`

**Acessibilidade:**
- `<header>` semântico
- `<nav aria-label="Breadcrumb">` com `<ol>` + `<li>`
- `aria-current="page"` no último item do breadcrumb
- `<h1>` para o título (somente um por página — o PageHeader emite o H1)

### 6. `DataTable` — tabela de dados com paginação

**Props:**
```ts
interface DataTableColumn<T> {
  key: keyof T | string      // chave do objeto de dado
  header: string             // texto do cabeçalho
  render?: (value: unknown, row: T) => React.ReactNode   // renderização customizada
  width?: string             // ex: '120px', '20%'
}

interface DataTableProps<T extends object> {
  columns: DataTableColumn<T>[]
  data: T[]
  pageSize?: number           // padrão: 10
  emptyMessage?: string       // padrão: 'Nenhum registro encontrado'
  loading?: boolean           // exibe Skeleton nas rows quando true
  className?: string
}
```

**Comportamento:**
- Paginação: calculada localmente (sem `onPageChange` — dados chegam completos)
- Estado de paginação: `useState` interno com página atual (começa em 1)
- Slice de dados: `data.slice((page - 1) * pageSize, page * pageSize)`
- `loading=true`: renderizar `pageSize` rows de Skeleton (usa `<Skeleton>` átomo da Story 0.3b)
- `emptyMessage`: exibido em `<td colSpan={columns.length}>` quando `data.length === 0` e não `loading`
- `render` customizado: quando fornecido, chama `render(row[key], row)` — senão, `String(row[key])`
- Paginador: botões "Anterior" / "Próximo" via `<Button>` átomo + display "Página X de Y"
- Sem sorting nesta story — apenas exibição e paginação

**Estrutura HTML:**
```html
<div class="overflow-hidden rounded-lg border border-neutral-200">
  <div class="overflow-x-auto">
    <table class="w-full text-sm">
      <thead class="bg-neutral-50 border-b border-neutral-200">
        <tr>
          <th>...</th>
        </tr>
      </thead>
      <tbody class="divide-y divide-neutral-100 bg-white">
        <tr>
          <td>...</td>
        </tr>
      </tbody>
    </table>
  </div>
  <div class="pagination">...</div>
</div>
```

**Tokens a usar:**
- Header: `bg-neutral-50 text-neutral-500 font-medium text-xs uppercase tracking-wide`
- Cell: `text-neutral-900 py-3 px-4`
- Row hover: `hover:bg-neutral-50`
- Paginador: `bg-white border-t border-neutral-200 px-4 py-3 flex items-center justify-between`

**Acessibilidade:**
- `<table>` com `role="table"` (implícito, mas explícito para clareza)
- `<th scope="col">` em todos os cabeçalhos
- `<caption>` opcional — não obrigatório nesta story
- Paginador: `aria-label="Paginação"` no wrapper div

## Notas técnicas

### DataTable com generics

```tsx
function DataTable<T extends object>({ columns, data, ...props }: DataTableProps<T>) {
  // ...
}
```

Vitest + @testing-library/react suporta componentes genéricos — usar `as DataTableProps<{id: number, name: string}>` no teste.

### SidebarNav e collapsed

O estado `collapsed` é **controlado externamente** — SidebarNav não mantém estado interno de collapsed. O componente pai (ex: AppShell) gerencia via useState:

```tsx
const [collapsed, setCollapsed] = useState(false)
<AppShell sidebar={<SidebarNav collapsed={collapsed} onToggleCollapse={() => setCollapsed(c => !c)} items={...} />}>
```

Isso permite que o módulo persista o estado (localStorage, Zustand) sem SidebarNav saber.

### DataTable e Skeleton

Quando `loading=true`, renderizar `pageSize` rows com células contendo `<Skeleton height="1rem" />`. Usar `Array.from({ length: pageSize })` para gerar as rows fantasma.

### AppShell e overflow

`<div class="flex h-screen overflow-hidden">` no root garante que apenas o `<main>` rola, não a página toda. O `<aside>` permanece fixo verticalmente.

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| DataTable generics causar problemas com Vitest type checking | Médio | Usar `as` type assertion nos testes; verificar tsconfig `strict` |
| SidebarNav transition CSS não suave em todos os browsers | Baixo | Testar `transition-all` vs `transition-width` — fallback para `transition-all` |
| `h-screen` no AppShell causar problemas em ambientes com iframe | Baixo | Aceitar limitação — AppShell é para uso em top-level de módulo |
| Paginação DataTable com data vazio causar divisão por zero | Médio | Guard: `Math.max(1, Math.ceil(data.length / pageSize))` para total de páginas |
| SidebarNav com item sem `href` (seções sem link) | Baixo | Aceitar que esta story require `href` sempre — seções expandíveis fora do escopo |
