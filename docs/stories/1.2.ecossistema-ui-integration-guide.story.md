## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema (Estética IA, Auth, Gestão ou ML Lab),
**quero** um guia de integração completo e um mapa de componentes,
**para que** eu consiga instalar e configurar `@aiox/ui` do zero em menos de 15 minutos e saiba exatamente qual componente usar em cada situação.

## Contexto

Com o build de distribuição (Story 1.0) e as composições de página (Story 1.1), o `@aiox/ui` está tecnicamente pronto para uso. O que falta é documentação de integração: como instalar, como configurar o Tailwind, quais componentes existem e como migrar telas existentes.

Esta story cria 3 documentos em `docs/guides/`:

1. `integration.md` — instalação e configuração passo a passo
2. `component-map.md` — mapa de todos os 19 componentes com quando usar cada um
3. `migration-checklist.md` — checklist para migrar HTML nativo para `@aiox/ui`

**Não há código a implementar** — apenas documentação. Critério de aceitação: os 3 arquivos existem e têm conteúdo completo conforme especificado.

```md
## Uso básico

import { Button, FormField, Input } from '@aiox/ui'

function LoginForm() {
  return (
    <form>
      <FormField label="E-mail">
        <Input type="email" placeholder="seu@email.com" />
      </FormField>
      <Button variant="primary" type="submit">
        Entrar
      </Button>
    </form>
  )

```

#### Seção 5 — Layout completo com AppShell
```md
## Layout completo

import { AppShell, SidebarNav, PageHeader } from '@aiox/ui'

const NAV_ITEMS = [
  { label: 'Dashboard', icon: 'LayoutDashboard', href: '/dashboard', active: true },
  { label: 'Clientes', icon: 'Users', href: '/clientes' },
]

function AppLayout({ children }) {
  const [collapsed, setCollapsed] = useState(false)

  return (
    <AppShell
      sidebar={
        <SidebarNav
          items={NAV_ITEMS}
          collapsed={collapsed}
          onToggleCollapse={() => setCollapsed(c => !c)}
        />
      }
    >
      <PageHeader title="Dashboard" />
      {children}
    </AppShell>
  )
}
```

#### Seção 8 — Troubleshooting
```md
## Troubleshooting

### Classes Tailwind não aplicadas
Certifique-se de incluir o path do pacote no `content` do `tailwind.config.js`:
`'./node_modules/@aiox/ui/src/**/*.{ts,tsx}'`

### Erro: Cannot find module '@aiox/ui'
Execute `npm install` novamente. Se instalado via GitHub, verifique a conectividade e
que o branch `main` existe: `github:ewertonfm00/ecossistema-ui#main`

### TypeScript não reconhece os tipos
Verifique que `@aiox/ui` em `node_modules` tem `dist/index.d.ts`. Se instalado de `src/`
diretamente (versão antiga), execute `npm run build` no ecossistema-ui e reinstale.
```

### 3. `docs/guides/component-map.md` — conteúdo obrigatório

Mapa visual de todos os 19 componentes do `@aiox/ui` com quando usar cada um.

**Estrutura obrigatória:**

```md
# Mapa de Componentes — @aiox/ui

> 19 componentes disponíveis | Versão 0.1.0

## Átomos (11)

| Componente | Import | Quando usar | Props principais |
|------------|--------|-------------|-----------------|
| `Button` | `import { Button }` | Toda ação do usuário que dispara operação | `variant`, `size`, `disabled`, `loading` |
| `Input` | `import { Input }` | Campos de texto, email, senha, número | `type`, `placeholder`, `error`, `disabled` |
| `Label` | `import { Label }` | Rótulo associado a um campo de formulário | `htmlFor`, `required` |
| `Textarea` | `import { Textarea }` | Texto longo, observações, descrições | `rows`, `placeholder`, `error` |
| `Select` | `import { Select }` | Seleção de opção em lista curta (≤10 itens) | `options`, `value`, `onChange`, `error` |
| `Checkbox` | `import { Checkbox }` | Seleção múltipla ou consentimento | `checked`, `onChange`, `disabled` |
| `Radio` | `import { Radio }` | Seleção exclusiva entre opções visíveis | `name`, `value`, `checked`, `onChange` |
| `Switch` | `import { Switch }` | Toggle de configuração (on/off) | `checked`, `onChange`, `disabled` |
| `Badge` | `import { Badge }` | Status, categoria, contagem | `variant`, `size` |
| `Skeleton` | `import { Skeleton }` | Placeholder durante carregamento | `height`, `width`, `className` |
| `Icon` | `import { Icon }` | Ícones Lucide em qualquer contexto | `name`, `size`, `className` |

## Moléculas (4)

| Componente | Import | Quando usar | Props principais |
|------------|--------|-------------|-----------------|
| `FormField` | `import { FormField }` | Container de campo com label + mensagem de erro | `label`, `error`, `required`, `children` |
| `Alert` | `import { Alert }` | Mensagem de feedback contextual (info, sucesso, erro, aviso) | `variant`, `children`, `onDismiss` |
| `Card` | `import { Card }` | Container de conteúdo com borda e sombra sutil | `className`, `children` |
| `Modal` | `import { Modal }` | _Planejado — Épico 04_ | — |

**Nota:** `Modal` é planejado no Épico 04 — ainda não disponível.

## Organismos (4)

| Componente | Import | Quando usar | Props principais |
|------------|--------|-------------|-----------------|
| `AppShell` | `import { AppShell }` | Layout root de toda tela autenticada | `sidebar`, `children` |
| `SidebarNav` | `import { SidebarNav }` | Navegação lateral colapsável | `items`, `collapsed`, `onToggleCollapse`, `logo` |
| `PageHeader` | `import { PageHeader }` | Cabeçalho de página com título, breadcrumb e ações | `title`, `breadcrumbs`, `actions` |
| `DataTable` | `import { DataTable }` | Listagem de dados com paginação local | `columns`, `data`, `pageSize`, `loading` |

## Decisão de uso rápida

| Cenário | Componentes |
|---------|------------|
| Tela de login | `div` centrado + `Card` + `FormField` + `Input` + `Button` |
| Dashboard | `AppShell` + `SidebarNav` + `PageHeader` + `Card` (×n métricas) |
| Listagem de dados | `AppShell` + `SidebarNav` + `PageHeader` + `DataTable` |
| Formulário | `AppShell` + `SidebarNav` + `PageHeader` + `Card` + `FormField` + inputs |
| Empty state | `Card` + `Icon` + `Alert` + `Skeleton` (quando loading) |
| Status em célula de tabela | `Badge` com `variant="success"/"error"/"warning"/"neutral"` |
| Loading de seção | `Skeleton` com `height` e `width` adequados |
```

### 4. `docs/guides/migration-checklist.md` — conteúdo obrigatório

Checklist para migrar telas existentes nos módulos consumidores para `@aiox/ui`.

**Estrutura obrigatória:**

```md
# Migration Checklist — Adotando @aiox/ui nos Módulos

## Antes de começar

- [ ] `@aiox/ui` instalado: `npm install github:ewertonfm00/ecossistema-ui#main`
- [ ] `tailwind.config.js` configurado com o preset (ver `docs/guides/integration.md`)
- [ ] Testes da tela passando antes da migração (linha base)

## Elementos HTML → Componentes @aiox/ui

### Botões
- [ ] `<button className="...">` → `<Button variant="primary|secondary|ghost">`
- [ ] `<button disabled>` → `<Button disabled>` (mantém acessibilidade automaticamente)
- [ ] `<button type="submit">` → `<Button type="submit" variant="primary">`

### Campos de formulário
- [ ] `<label>` + `<input>` soltos → `<FormField label="..."><Input /></FormField>`
- [ ] `<input type="text">` → `<Input type="text" />`
- [ ] `<input type="email">` → `<Input type="email" />`
- [ ] `<input type="password">` → `<Input type="password" />`
- [ ] `<textarea>` → `<Textarea rows={n} />`
- [ ] `<select>` → `<Select options={[{ value, label }]} />`
- [ ] `<input type="checkbox">` → `<Checkbox />`
- [ ] `<input type="radio">` → `<Radio name="..." value="..." />`
- [ ] Toggle/switch customizado → `<Switch />`
- [ ] Mensagem de erro inline → `<FormField error="mensagem">` (eliminar div de erro manual)

### Layout
- [ ] `<div className="flex h-screen">` (layout com sidebar) → `<AppShell sidebar={...}>`
- [ ] `<aside>` de navegação customizada → `<SidebarNav items={...} />`
- [ ] `<header>` de página com título + botões → `<PageHeader title="..." actions={...}>`
- [ ] Breadcrumb manual → `PageHeader breadcrumbs={[...]}`

### Tabelas
- [ ] `<table>` com paginação manual → `<DataTable columns={...} data={...} pageSize={10}>`
- [ ] Loading state com spinner → `<DataTable loading={true}>` (usa Skeleton automaticamente)
- [ ] "Sem dados" manual → `<DataTable emptyMessage="Nenhum registro encontrado">`

### Feedback e status
- [ ] `<div className="alert alert-danger">` → `<Alert variant="error">`
- [ ] `<div className="alert alert-success">` → `<Alert variant="success">`
- [ ] `<div className="alert alert-warning">` → `<Alert variant="warning">`
- [ ] `<div className="alert alert-info">` → `<Alert variant="info">`
- [ ] Badge de status customizado → `<Badge variant="success|error|warning|neutral">`
- [ ] Skeleton de loading customizado → `<Skeleton height="..." width="..." />`

### Ícones
- [ ] Qualquer biblioteca de ícones em uso → `<Icon name="..." />` (Lucide — verificar nome no site lucide.dev)

## Após a migração

- [ ] `npm test -- --run` → todos os testes passando
- [ ] Verificar visualmente no browser: cores, espaçamento, tipografia consistentes com design system
- [ ] Remover imports de CSS/classes Tailwind hardcoded que foram substituídos
- [ ] Remover dependências de UI que não são mais necessárias (ex: react-icons, componentes de UI customizados inline)
- [ ] Abrir PR — `@devops` (Gage) faz o push

## Escopo negativo — não migrar nesta fase

- ❌ Validação de formulários (React Hook Form + Zod) — permanece no módulo
- ❌ Gerenciamento de estado (Zustand, Context) — permanece no módulo
- ❌ Chamadas de API e fetch — permanece no módulo
- ❌ Animações customizadas além do que os componentes oferecem — avaliar caso a caso
- ❌ Componentes específicos do domínio (ex: CalendarPicker de agendamentos) — usar Base + customizar
```

## Notas técnicas

### Por que Markdown e não MDX/Storybook Docs?

Os guias são consumidos por devs nos módulos consumidores, não apenas por quem usa o Storybook. Markdown em `docs/guides/` é acessível diretamente no repositório GitHub, sem precisar rodar o Storybook.

### Tabela de 19 componentes — contagem

Átomos (Story 0.2 + 0.3b): Button, Input, Label, Textarea, Select, Checkbox, Radio, Switch, Badge, Skeleton, Icon = **11**
Moléculas (Story 0.3): FormField, Alert, Card + Modal planejado = **3 disponíveis + 1 planejado**
Organismos (Story 0.4): AppShell, SidebarNav, PageHeader, DataTable = **4**
Total disponível: **18** | Total incluindo planejado: 19

**Decisão:** o `component-map.md` documenta 18 componentes disponíveis e menciona Modal como "planejado — Épico 04". O título pode dizer "19 componentes" se contar o planejado, ou "18 componentes disponíveis" para precisão — `@dev` escolhe.
