## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema,
**quero** um componente `Stepper` com indicação visual de progresso multi-step (estados completed/current/pending, orientações horizontal e vertical),
**para que** eu possa construir wizards e fluxos de cadastro multi-etapa sem reimplementar lógica de progresso em cada módulo e sem divergência visual.

---

## Contexto

Nenhum módulo do ecossistema tem um componente de wizard padronizado. Cadastro de paciente na Estética IA, onboarding no AIOX Gestão e configuração de experimento no ML Lab compartilham o mesmo padrão de progresso multi-step — sem `Stepper` no design system, cada módulo criará o próprio, com divergência garantida.

**Decisão arquitetural:** Compound component com Context API (padrão já estabelecido em Modal/Drawer e Tabs). `<Stepper>` provê contexto; `<StepList>`, `<Step>` e `<StepPanel>` são consumidores. Sem dependências externas. Estado de cada step (completed/current/pending) derivado automaticamente do `currentStep` — o dev não precisa passar estado manualmente para cada `<Step>`. Suporte a modo controlado (`value` + `onChange`) e não-controlado (`defaultValue`).

**Classificação:** Molécula — composição de Context API + sub-componentes sem lógica de página. Vai para `src/molecules/Stepper/`.

### 2. Interface pública

```tsx
// Stepper — provider do contexto
interface StepperProps {
  value?: number            // modo controlado (índice 0-based do step atual)
  defaultValue?: number     // modo não-controlado (padrão: 0)
  onChange?: (step: number) => void
  orientation?: 'horizontal' | 'vertical'  // padrão: 'horizontal'
  className?: string
  children: React.ReactNode
}

// StepList — container visual dos indicadores de step
interface StepListProps {
  children: React.ReactNode
  className?: string
}

// Step — indicador individual de step
interface StepProps {
  label: string             // texto do step (obrigatório)
  description?: string      // subtítulo opcional exibido abaixo do label
  icon?: React.ReactNode    // ícone customizado no círculo (substitui número/check)
  className?: string
}

// StepPanel — painel de conteúdo associado ao step atual
interface StepPanelProps {
  step: number              // índice 0-based do step que este painel representa
  children: React.ReactNode
  className?: string
}
```

### 3. Comportamento e estados

#### 3.1 Estado dos steps (derivado automaticamente)

O estado de cada `<Step>` é calculado pelo Stepper — o desenvolvedor NÃO precisa passar estado manualmente:

| Condição | Estado | Visual |
|----------|--------|--------|
| `index < currentStep` | `completed` | Círculo com ícone `Check` (lucide), `bg-primary-600 text-white` |
| `index === currentStep` | `current` | Círculo com número, `bg-primary-600 text-white`, `ring-2 ring-primary-300 ring-offset-2` |
| `index > currentStep` | `pending` | Círculo com número, `bg-white border-2 border-neutral-300 text-neutral-400` |

#### 3.2 Modo controlado vs não-controlado

| Prop | Comportamento |
|------|--------------|
| `value` + `onChange` | Controlado externamente — componente não gerencia estado interno |
| `defaultValue` | Não-controlado — inicializa com o valor passado (padrão: `0`) |
| nenhum dos dois | Não-controlado — inicializa no step `0` |

#### 3.3 Orientação

| Orientação | Visual |
|-----------|--------|
| `horizontal` (padrão) | Steps em linha horizontal (`flex-row`), conectores horizontais entre círculos |
| `vertical` | Steps em coluna (`flex-col`), conectores verticais à esquerda dos rótulos |

#### 3.4 Conectores visuais

A linha que conecta dois steps reflete o estado do step **anterior**:
- Após step completado → linha sólida `bg-primary-600`
- Após step atual ou pendente → linha tracejada / `bg-neutral-200`

Para `horizontal`: linha `h-0.5 flex-1 mx-2` entre círculos.
Para `vertical`: linha `w-0.5 flex-1 my-1 ml-[calc(1.25rem-1px)]` abaixo de cada círculo (exceto o último).

#### 3.5 Clicabilidade dos steps

| Comportamento | Configuração |
|--------------|-------------|
| Padrão | Steps `completed` são **clicáveis** (permitem voltar). Steps `pending` são **não-clicáveis**. |
| Step `current` | Não-clicável (já está aqui). |

Clique em step completado chama `setCurrentStep(index)` — navega de volta para aquele step.

#### 3.6 StepPanel

`<StepPanel step={n}>` renderiza seu conteúdo apenas quando `currentStep === n`, usando `hidden` do Tailwind (não desmonta DOM):

```tsx
<div className={cn('outline-none', currentStep !== step && 'hidden', className)}>
  {children}
</div>
```

#### 3.7 ARIA obrigatório

```tsx
// StepList — nav container
<nav aria-label="Progresso">
  <ol className="flex ...">

// Step — list item + button
<li>
  <button
    aria-current={isCurrent ? 'step' : undefined}
    aria-label={`${label}${isCompleted ? ' — concluído' : isCurrent ? ' — atual' : ' — pendente'}`}
    disabled={isPending}   // steps pendentes não são interativos
    onClick={() => isCompleted && setCurrentStep(index)}
  >
    {/* círculo com número ou ícone check */}
    <span aria-hidden="true">{isCompleted ? <Check /> : index + 1}</span>
  </button>
  <span>{label}</span>
  {description && <span className="text-xs text-neutral-500">{description}</span>}
</li>
```

**IDs gerados via `useId()` no nível do `<Stepper>` — evita colisões em múltiplas instâncias na mesma página.**

## Notas técnicas para @dev

- **Context API:** `StepperContext` com `{ currentStep, setCurrentStep, orientation, totalSteps, idPrefix }` — mesma abordagem de `TabsContext`
- **useId():** gerar `idPrefix` único no nível do `<Stepper>` — compor IDs como `${idPrefix}-step-${index}` e `${idPrefix}-panel-${index}`
- **Estado derivado:** cada `<Step>` recebe seu `index` via React.Children ou via contexto + contador interno. **Abordagem recomendada:** usar `React.Children.map` no `<StepList>` para injetar `index` como prop via `React.cloneElement`, e contar `totalSteps` — evita ter que passar `index` manualmente
- **`React.cloneElement` + `React.Children.map`:** aplicar no `StepList` para injetar `index` e separador visual (conector) automaticamente entre steps:
  ```tsx
  const children = React.Children.toArray(childrenProp) as React.ReactElement<StepProps & { index: number }>[]
  // ou via contador de contexto com useContext + increment pattern
  ```
  **Alternativa mais simples (recomendada):** Cada `<Step>` usa um contador de contexto incrementado em cada render — `StepperContext` expõe `registerStep()` que retorna o próximo índice
- **Classe `hidden`:** `<StepPanel>` usa `hidden` para ocultar painéis inativos — não desmonta DOM
- **Linhas conectoras:** renderizadas dentro do `<StepList>` como separadores entre steps, com largura/altura dinâmica baseada em orientação
- **TypeScript:** `noUncheckedIndexedAccess: true` ativo — todo acesso a array por índice usa optional chaining
- **Controlled mode:** `const isControlled = value !== undefined` — aliased condition narrowing (TypeScript 4.4+)

### Alternativa para injeção de índice (mais simples)

Usar `React.Children.map` no `StepList`:

```tsx
export function StepList({ children, className }: StepListProps) {
  const { orientation } = useStepperContext()
  const steps = React.Children.toArray(children)
  const total = steps.length
  return (
    <ol className={cn('flex', orientation === 'horizontal' ? 'flex-row items-center' : 'flex-col', className)}>
      {steps.map((child, index) => (
        <React.Fragment key={index}>
          {React.cloneElement(child as React.ReactElement<StepInternalProps>, { index, total })}
          {index < total - 1 && <StepConnector completed={index < currentStep} orientation={orientation} />}
        </React.Fragment>
      ))}
    </ol>
  )
}
```

Isso elimina a necessidade de `registerStep()` e mantém o componente stateless por step.

### Observações

**LOW (não bloqueante):** `index?: number` exposto na interface pública de `StepProps` com `@internal` em JSDoc. Usuário poderia passar `index` manualmente e interferir na auto-injeção via `React.cloneElement`. Risco baixo — padrão estabelecido em compound components React. Não impacta comportamento correto no uso normal.

**LOW (não bloqueante):** `StepPanel` sem `tabIndex={0}`. Diferente de `TabPanel` em Tabs, os painéis de Stepper normalmente contêm formulários com elementos focáveis nativos. Omissão intencional e aceitável para este componente.
