# Story 8.0 — Épico 10 — Feedback & Status (ProgressBar, StepIndicator, EmptyState)

## Status: Ready for Review

## Story

**As a** desenvolvedor integrando componentes nos módulos do ecossistema
**I want** componentes padronizados de feedback visual (ProgressBar, StepIndicator, EmptyState)
**So that** posso comunicar progresso de tarefas, etapas de fluxo e estados vazios com consistência visual sem duplicar implementações entre módulos

## Background

O ecossistema precisa de padrões de feedback visual para:
- **ProgressBar**: barra de progresso para uploads, carregamento de dados, metas de faturamento
- **StepIndicator**: indicador visual de etapas (somente display — diferente do `Stepper` interativo do Épico 07)
- **EmptyState**: estado vazio padronizado para listas sem dados, resultados sem busca, etc.

O `Stepper` (Story 5.1) é um wizard interativo com navegação. O `StepIndicator` desta story é puramente visual — recebe `current` e `steps` e renderiza sem onClick.

## Acceptance Criteria

### AC1 — ProgressBar

- [ ] Aceita: `value: number`, `max?: number` (padrão 100), `variant?: 'default' | 'success' | 'warning' | 'error'` (padrão `'default'`), `size?: 'sm' | 'md' | 'lg'` (padrão `'md'`), `label?: string`, `showValue?: boolean` (padrão false), `className?`
- [ ] Largura da barra interna = `(value / max) * 100%` (clamped 0-100)
- [ ] Variantes de cor:
  - `default` → `bg-primary-600`
  - `success` → `bg-green-500`
  - `warning` → `bg-amber-500`
  - `error` → `bg-red-500`
- [ ] Tamanhos (altura da track):
  - `sm` → `h-1.5`
  - `md` → `h-2.5`
  - `lg` → `h-4`
- [ ] `label` renderizado acima da barra (texto `text-sm text-neutral-600`)
- [ ] `showValue=true` exibe percentual calculado à direita da label (e.g., "67%")
- [ ] ARIA: `role="progressbar"`, `aria-valuenow={value}`, `aria-valuemin={0}`, `aria-valuemax={max}`, `aria-label={label ?? 'Progresso'}`
- [ ] Track de fundo: `bg-neutral-200 rounded-full overflow-hidden`
- [ ] Transição suave: `transition-all duration-300` na barra interna

### AC2 — StepIndicator (visual apenas)

- [ ] Aceita: `steps: StepIndicatorStep[]`, `current: number`, `orientation?: 'horizontal' | 'vertical'` (padrão `'horizontal'`), `className?`
- [ ] Interface `StepIndicatorStep = { label: string; description?: string }`
- [ ] Renderiza lista de steps como `<ol>` com `aria-label="Indicador de etapas"`
- [ ] Estados visuais baseados em `current`:
  - `index < current`: completed — círculo `bg-primary-600` com ícone `Check`
  - `index === current`: current — círculo `bg-primary-600` com número, ring externo `ring-2 ring-primary-300`
  - `index > current`: pending — círculo `bg-white border-2 border-neutral-300 text-neutral-400`
- [ ] Conector entre steps (linha horizontal em `horizontal`, vertical em `vertical`)
  - Concluído: `bg-primary-600`
  - Pendente: `bg-neutral-200`
- [ ] **Nenhum `onClick`** — puramente display (diferença fundamental do Stepper)
- [ ] Steps são `<li>` com role implícito de list item (não button, não link)
- [ ] `description` exibida abaixo do label quando fornecida (texto `text-xs text-neutral-500`)
- [ ] Orientação horizontal: `flex-row items-start`, conector `h-0.5 flex-1`
- [ ] Orientação vertical: `flex-col`, conector `w-0.5 h-8 ml-[19px]`

### AC3 — EmptyState

- [ ] Aceita: `title: string`, `description?: string`, `icon?: React.ReactNode`, `action?: EmptyStateAction`, `size?: 'sm' | 'md' | 'lg'` (padrão `'md'`), `className?`
- [ ] Interface `EmptyStateAction = { label: string; onClick: () => void }`
- [ ] Layout: `flex flex-col items-center text-center gap-4` dentro de um `div`
- [ ] `icon` renderizado em container circular neutro (escala conforme `size`)
- [ ] `title` em `text-neutral-800 font-medium`
- [ ] `description` em `text-neutral-500 text-sm`
- [ ] `action` renderiza `<Button variant="primary">` com `onClick` fornecido
- [ ] Tamanhos:
  - `sm`: ícone 32px, title `text-sm`, padding `py-6`
  - `md`: ícone 48px, title `text-base`, padding `py-10`
  - `lg`: ícone 64px, title `text-lg`, padding `py-16`
- [ ] ARIA: `role="status"` no container raiz, `aria-label={title}`

### AC4 — Tipagem TypeScript Strict

- [ ] `StepIndicatorStep` e `EmptyStateAction` exportados
- [ ] Sem `any`, sem `!` non-null assertion
- [ ] `value / max * 100` com guard contra divisão por zero (`max === 0` → 0)

### ProgressBar — Cálculo de Largura

```tsx
const pct = max > 0 ? Math.min(100, Math.max(0, (value / max) * 100)) : 0
// <div style={{ width: `${pct}%` }} ... />
```

### StepIndicator vs Stepper — Diferença Fundamental

| Característica | Stepper (5.1) | StepIndicator (8.0) |
|----------------|----------------|----------------------|
| Interativo | Sim (onClick nos steps) | Não (somente display) |
| Compound component | Sim | Não |
| API | Context + filhos JSX | Props diretas (steps, current) |
| StepPanel | Sim | Não |
| Caso de uso | Wizard navegável | Indicador de progresso read-only |

### EmptyState — Integração com Button

```tsx
import { Button } from '../../atoms/Button'
// ...
{action && <Button variant="primary" onClick={action.onClick}>{action.label}</Button>}
```

const iconSize = { sm: 32, md: 48, lg: 64 }[size]

## Dev Notes

- StepIndicator usa mesmos valores de estilo do Stepper (ml-[19px] no conector vertical) — isso é intencional para consistência visual entre os dois componentes
- EmptyState depende do componente Button existente em `src/atoms/Button` — importar diretamente, não via index
- `React.cloneElement` no ícone do EmptyState: usar com cautela — verificar se o elemento é válido com `React.isValidElement(icon)` antes de clonar. Se não for (e.g., string ou null), renderizar diretamente sem clonar.
