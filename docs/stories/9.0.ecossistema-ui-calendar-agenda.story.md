# Story 9.0 — Épico 11 — Calendar & Agenda (CalendarPicker, DateRangePicker, AgendaView)

## Status: Draft

## Story

**As a** desenvolvedor construindo funcionalidades de agendamento nos módulos do ecossistema
**I want** componentes de calendário standalone (CalendarPicker, DateRangePicker, AgendaView)
**So that** posso integrar seleção de datas e visualização de agenda de procedimentos sem dependências de bibliotecas de calendário externas

## Background

Os módulos do ecossistema precisam de componentes de agenda para:
- **CalendarPicker**: seleção de data standalone (month view) para agendamento de consultas
- **DateRangePicker**: seleção de intervalo de datas para filtros de período (relatórios, disponibilidade)
- **AgendaView**: visualização semanal de procedimentos agendados com slots de horário

O `DatePicker` existente (Épico 04) é um componente de formulário (input + dropdown calendar). Os componentes desta story são standalone — usados em contextos de exibição e seleção visual, não em campos de formulário inline.

Implementação puramente em JavaScript/Date — sem `date-fns`, `dayjs`, `luxon` ou similares. Locale suportado: `'pt-BR'` (padrão) e `'en-US'`.

## Acceptance Criteria

### AC1 — CalendarPicker

- [ ] Aceita: `value?: Date` (controlado), `defaultValue?: Date`, `onChange: (date: Date) => void`, `disabled?: (date: Date) => boolean`, `locale?: 'pt-BR' | 'en-US'` (padrão `'pt-BR'`), `className?`
- [ ] Renderiza view de mês com header: `"<" [Mês Ano] ">"`
- [ ] Header exibe nome do mês e ano usando `Intl.DateTimeFormat` com o locale
- [ ] Botões `<` e `>` navegam entre meses (prev/next)
- [ ] Grid de 7 colunas com header de dias da semana (`Dom Seg Ter Qua Qui Sex Sáb` para pt-BR)
- [ ] Células dos dias do mês — regras:
  - Dias de outros meses: cor `text-neutral-300`, não clicáveis
  - Dia selecionado (`value`): `bg-primary-600 text-white rounded-full`
  - Hoje (`Date === new Date()`): `ring-1 ring-primary-400`
  - `disabled(date) === true`: `text-neutral-300 cursor-not-allowed`, não dispara `onChange`
  - Outros dias: `hover:bg-neutral-100 rounded-full cursor-pointer`
- [ ] Clique em dia válido chama `onChange(date)` com midnight no fuso local
- [ ] Uncontrolled: quando `value` não fornecido, usa `defaultValue` + estado interno `internalDate`
- [ ] Navegação por teclado:
  - `ArrowLeft/Right` move entre dias
  - `ArrowUp/Down` move 7 dias (linha)
  - `Enter/Space` seleciona o dia focado
  - `PageUp/PageDown` navega entre meses
- [ ] ARIA: `role="grid"` no container do grid, `role="gridcell"` em cada célula, `aria-label` com dia por extenso no gridcell, `aria-selected` na data selecionada, `aria-disabled` em datas desabilitadas

### AC2 — DateRangePicker

- [ ] Aceita: `value?: DateRange` (controlado), `defaultValue?: DateRange`, `onChange: (range: DateRange) => void`, `disabled?: (date: Date) => boolean`, `showTwoMonths?: boolean` (padrão true), `locale?: 'pt-BR' | 'en-US'` (padrão `'pt-BR'`), `className?`
- [ ] Interface `DateRange = { start: Date | null; end: Date | null }`
- [ ] Renderiza 1 ou 2 meses lado a lado (conforme `showTwoMonths`)
- [ ] Primeiro clique define `start`, segundo clique define `end`
- [ ] Se `start` já definido e segundo clique for antes de `start`: inverte (novo start = clicado, end = antigo start)
- [ ] Enquanto `start` está definido mas `end` não: células entre `start` e cursor do mouse ganham highlight `bg-primary-50` (hover range preview)
- [ ] Intervalo definido (`start` e `end`):
  - `start` e `end`: `bg-primary-600 text-white rounded-full`
  - Dias entre eles: `bg-primary-100 text-primary-800` (sem rounded no meio, apenas nas extremidades)
- [ ] Navegação de meses: botões prev/next controlam ambos os meses em sincronia
- [ ] ARIA: `aria-label` em cada célula indicando se está no intervalo, selecionada como start/end, ou disponível

### AC3 — AgendaView

- [ ] Aceita: `events: AgendaEvent[]`, `currentDate: Date`, `onEventClick?: (event: AgendaEvent) => void`, `startHour?: number` (padrão 8), `endHour?: number` (padrão 20), `className?`
- [ ] Interface:
  ```ts
  interface AgendaEvent {
    id: string
    title: string
    start: Date
    end: Date
    color?: string
    description?: string
  }
  ```
- [ ] Exibe 7 dias a partir de `currentDate` (segunda-feira da semana de `currentDate`)
- [ ] Colunas: 7 (um por dia), identificadas pelo dia da semana + data
- [ ] Linhas: slots de 1 hora de `startHour` até `endHour`
- [ ] Cada evento renderizado como card posicionado na grade:
  - Posicionamento vertical: `top` calculado como `(event.start.getHours() - startHour) * SLOT_HEIGHT + minuteOffset`
  - Altura: `(duration_in_minutes / 60) * SLOT_HEIGHT`
  - Largura: 90% da coluna
  - Cor de fundo: `event.color` ou `primary-100` como fallback
  - Trunca título com `overflow-hidden text-ellipsis`
- [ ] `onEventClick` chamado com o evento ao clicar no card
- [ ] Eventos em dias fora do intervalo `startHour-endHour` são exibidos no topo/fundo da coluna com aviso
- [ ] Linha de "hora atual" visível quando `currentDate` inclui o dia de hoje (`position: absolute`, `border-t-2 border-red-400`)
- [ ] Scroll vertical interno quando conteúdo > viewport (container `overflow-y-auto`)
- [ ] ARIA: `role="grid"`, eventos com `role="button"` quando `onEventClick` definido, `aria-label` com título + horário do evento

### Geração do Grid de Dias (CalendarPicker)

```ts
function generateCalendarDays(year: number, month: number): Date[] {
  const firstDay = new Date(year, month, 1)
  const startDate = startOfWeek(firstDay)   // domingo da semana que contém o 1º
  const days: Date[] = []
  for (let i = 0; i < 42; i++) {            // sempre 6 semanas = 42 células
    days.push(addDays(startDate, i))
  }
  return days
}
```

Grid 7×6 = 42 células, sempre completo (dias de outros meses preenchidos).

### DateRangePicker — Estado de Seleção Parcial

```ts
// Estado interno quando controlado externamente:
const [selectionPhase, setSelectionPhase] = useState<'none' | 'selecting'>('none')
const [hoverDate, setHoverDate] = useState<Date | null>(null)

// Ao clicar em uma data:
function handleDateClick(date: Date) {
  if (selectionPhase === 'none' || !effectiveValue.start) {
    onChange({ start: date, end: null })
    setSelectionPhase('selecting')
  } else {
    const start = effectiveValue.start
    if (date < start) {
      onChange({ start: date, end: start })
    } else {
      onChange({ start, end: date })
    }
    setSelectionPhase('none')
  }
}
```

### AgendaView — Posicionamento de Eventos

```ts
const SLOT_HEIGHT = 60  // pixels por hora

function eventStyle(event: AgendaEvent, startHour: number): React.CSSProperties {
  const startMinutes = (event.start.getHours() - startHour) * 60 + event.start.getMinutes()
  const endMinutes = (event.end.getHours() - startHour) * 60 + event.end.getMinutes()
  const duration = Math.max(30, endMinutes - startMinutes)  // mínimo 30min de altura
  return {
    position: 'absolute',
    top: `${(startMinutes / 60) * SLOT_HEIGHT}px`,
    height: `${(duration / 60) * SLOT_HEIGHT}px`,
    left: '2%',
    width: '96%',
  }
}
```

### Semana de Segunda-Feira

Para `startOfWeek`, retornar a segunda-feira:
```ts
function startOfWeek(date: Date): Date {
  const d = new Date(date)
  const day = d.getDay()  // 0=Dom, 1=Seg, ...
  const diff = day === 0 ? -6 : 1 - day
  d.setDate(d.getDate() + diff)
  d.setHours(0, 0, 0, 0)
  return d
}
```

## Dev Notes

- Esta é a story mais complexa do épico — AgendaView envolve posicionamento absoluto baseado em cálculos de tempo. Recomendar implementar na ordem: `calendar.utils.ts` → `CalendarPicker` → `DateRangePicker` → `AgendaView`
- CalendarPicker: a navegação por teclado com `onKeyDown` no `role="grid"` requer gerenciamento de `focusedDate` como estado separado de `selectedDate`
- DateRangePicker: o hover preview exige `onMouseEnter` em cada célula — atenção para não causar re-renders excessivos; usar `useCallback` nos handlers
- AgendaView: eventos sobrepostos no mesmo dia/hora não são tratados nesta story (fora do escopo). Documentar no Storybook que eventos sobrepostos ficam por cima uns dos outros
- Testes de keyboard do CalendarPicker: usar `userEvent.keyboard` da `@testing-library/user-event` para simular teclas com efeitos colaterais corretos (foco, handlers)
- `Intl.DateTimeFormat` está disponível em JSDOM — não precisa de polyfill
