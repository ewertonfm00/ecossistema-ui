## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema,
**quero** um componente `Tabs` com navegação por abas (compound component controlado/não-controlado, variantes e teclado completo),
**para que** eu possa organizar conteúdo em seções sem reimplementar navegação por abas em cada módulo e sem divergência visual.

---

## Contexto

Nenhum módulo do ecossistema tem ainda uma forma padronizada de navegar entre seções de conteúdo. Sem `Tabs` no design system, cada módulo criará sua própria versão — divergência visual garantida. Tabs é o componente de organização de conteúdo mais universal: aparece em fichas de paciente, dashboards analíticos, telas de configuração e páginas de detalhe.

**Decisão arquitetural:** Compound component com Context API (padrão já estabelecido em Modal/Drawer). `<Tabs>` provê contexto; `<TabList>`, `<Tab>`, `<TabPanel>` são consumidores. Sem dependências externas (Radix/Headless UI). Lógica de keyboard navigation 100% interna. Suporte a modo controlado (`value` + `onChange`) e não-controlado (`defaultValue`).

**Classificação:** Molécula — mais complexo que um átomo isolado, mas sem lógica de layout de página. Vai para `src/molecules/Tabs/`.

### 2. Interface pública

```tsx
// Tabs — provider do contexto
interface TabsProps {
  value?: string           // modo controlado
  defaultValue?: string    // modo não-controlado (fallback para primeira tab se omitido)
  onChange?: (value: string) => void
  variant?: 'line' | 'pill' | 'card'   // padrão: 'line'
  className?: string
  children: React.ReactNode
}

// TabList — container das abas (role="tablist")
interface TabListProps {
  children: React.ReactNode
  className?: string
}

// Tab — aba individual (role="tab")
interface TabProps {
  value: string
  disabled?: boolean
  children: React.ReactNode
  className?: string
}

// TabPanel — painel de conteúdo (role="tabpanel")
interface TabPanelProps {
  value: string
  children: React.ReactNode
  className?: string
}
```

### 3. Comportamento e estados

#### 3.1 Modo controlado vs não-controlado

| Prop | Comportamento |
|------|--------------|
| `value` + `onChange` | Controlado externamente — componente não gerencia estado interno |
| `defaultValue` | Não-controlado — componente gerencia estado, inicializa com o valor passado |
| nenhum dos dois | Não-controlado — inicializa com a primeira tab não-desabilitada |

#### 3.2 Variantes visuais

| Variante | Visual |
|---------|--------|
| `line` (padrão) | Borda inferior na tab ativa (`border-b-2 border-primary-600`), fundo transparente |
| `pill` | Tab ativa com `bg-primary-50 text-primary-700 rounded-md`, sem borda |
| `card` | TabList com `bg-neutral-100 p-1 rounded-lg`, tab ativa com `bg-white shadow-sm rounded-md` |

#### 3.3 Estados da Tab

| Estado | Classes |
|--------|---------|
| Ativa | `text-primary-700` + indicador da variante |
| Inativa | `text-neutral-500 hover:text-neutral-700` |
| Desabilitada | `opacity-50 cursor-not-allowed` — não responde a clique ou teclado |
| Focus | `focus-visible:ring-2 focus-visible:ring-primary-500 focus-visible:ring-offset-1` |

#### 3.4 Keyboard navigation (ARIA Tabs Pattern)

| Tecla | Comportamento |
|-------|--------------|
| `ArrowRight` | Move foco para a próxima tab (wrap-around), pula desabilitadas |
| `ArrowLeft` | Move foco para a tab anterior (wrap-around), pula desabilitadas |
| `Home` | Move foco para a primeira tab não-desabilitada |
| `End` | Move foco para a última tab não-desabilitada |
| `Enter` / `Space` | Ativa a tab em foco |

#### 3.5 ARIA obrigatório

```tsx
// TabList
<div role="tablist" aria-label="...">

// Tab
<button
  role="tab"
  aria-selected={isActive}
  aria-controls={`panel-${value}`}
  id={`tab-${value}`}
  tabIndex={isActive ? 0 : -1}   // roving tabindex
>

// TabPanel
<div
  role="tabpanel"
  id={`panel-${value}`}
  aria-labelledby={`tab-${value}`}
  tabIndex={0}
>
```

**Nota:** IDs gerados via `useId()` + sufixo do `value` — evita colisões em múltiplas instâncias de `Tabs` na mesma página.

## Notas técnicas para @dev

- **Context API:** `TabsContext` com `{ activeValue, setActiveValue, variant, idPrefix }` — mesma abordagem do `ToastContext` em Story 2.1
- **useId():** gerar `idPrefix` único no nível do `<Tabs>` e compor IDs como `${idPrefix}-tab-${value}` e `${idPrefix}-panel-${value}`
- **roving tabindex:** tab ativa tem `tabIndex={0}`, todas as outras têm `tabIndex={-1}` — permite navegação com Tab do teclado para entrar no grupo, depois ArrowKeys dentro
- **Painel oculto:** usar `hidden` do Tailwind (`hidden` class) para ocultar painéis inativos — não desmonta o DOM (preserva estado interno dos formulários dentro do painel)
- **Wrap-around no teclado:** ao navegar além da última tab com ArrowRight, volta para a primeira; ao navegar antes da primeira com ArrowLeft, vai para a última
- **Tabs desabilitadas:** pular nas navegações por ArrowLeft/ArrowRight/Home/End — procurar o próximo habilitado na direção correta
- **TypeScript:** `noUncheckedIndexedAccess: true` ativo — todo acesso a array por índice deve usar optional chaining ou guard
