## Story

**Como** desenvolvedor implementando qualquer módulo do ecossistema (Auth, Estética IA, Marketing, Gestão, ML Lab),
**quero** um pacote `@aiox/ui` instalável como dependência git com os átomos core já implementados e o preset Tailwind exportado,
**para que** eu possa construir telas com consistência visual garantida sem recriar componentes do zero em cada módulo.

## Contexto

O `DESIGN.md` (v1, Story 0.1 ✅) define a especificação visual completa. Esta story cria a **implementação real** dessa spec: o repo `ecossistema-ui` com os 7 átomos de alta prioridade que todo módulo precisará desde a primeira tela.

Decisão de arquitetura (`@architect` Aria, 2026-06-13): design system global, instalado como:
```json
"@aiox/ui": "github:ewertonfm00/ecossistema-ui#main"
```

O módulo piloto (Estética IA — hub, M1) será o primeiro a consumir `@aiox/ui`, validando em produção antes dos demais.

**Stack obrigatória:** React 18 + TailwindCSS + TypeScript (padrão do ecossistema — `docs/architecture/01-arquitetura-ecossistema.md` §1)

**Ícones:** Lucide React (padrão do ecossistema — DESIGN.md §12 + §15)

## Critérios de aceitação

1. **Repo inicializado** — estrutura de diretórios conforme `DESIGN.md §14`:
   ```
   ecossistema-ui/
   ├── src/atoms/{Button,Input,Label,Badge,Avatar,Icon,Spinner}/
   │   ├── {Component}.tsx
   │   ├── {Component}.test.tsx
   │   └── index.ts
   ├── src/index.ts          # barrel export
   ├── tailwind-preset.js    # tokens do DESIGN.md §13
   ├── package.json          # name: "@aiox/ui"
   └── README.md
   ```

2. **`package.json`** com:
   - `name: "@aiox/ui"`
   - `peerDependencies`: `react ^18`, `react-dom ^18`, `tailwindcss ^3`
   - `dependencies`: `lucide-react ^0.x` (versão atual estável)
   - `devDependencies`: TypeScript, Vitest ou Jest + Testing Library

3. **`tailwind-preset.js`** exporta preset com todos os tokens do `DESIGN.md §13`:
   - `colors.primary` (11 tons teal)
   - `colors.accent` (rose-wine)
   - `colors.neutral` (slate)
   - `colors.semantic` (success/warning/error/info)
   - `fontFamily.sans` = Inter
   - `fontSize` (escala xs → display-2xl)
   - `borderRadius` default = 6px
   - `boxShadow` (elevation-xs, focus, focus-accent)
   - `zIndex` (raised → tooltip)

4. **Button** implementado conforme `DESIGN.md §16.1`:
   - 5 tamanhos: `xs` (h-7) / `sm` (h-9) / `md` (h-10) / `lg` (h-11) / `xl` (h-12)
   - 7 variantes: `primary` / `secondary` / `ghost` / `soft` / `destructive` / `destructive-ghost` / `link`
   - Estados: `disabled` (opacity-50) / `loading` (spinner 14px) / `icon-only` / `full-width`
   - `primary` usa `primary-700` como background (WCAG AA 5.48:1 — correção HC-1 do @qa)

5. **Input** implementado conforme `DESIGN.md §16.2`:
   - Altura base `h-10` (40px)
   - 3 tamanhos: `sm` (h-9) / `md` (h-10, default) / `lg` (h-11)
   - Estados: `default` / `error` (border-accent-600) / `disabled` (opacity-50)
   - Suporte a addon esquerdo/direito e ícone esquerdo/direito

6. **Label** implementado conforme `DESIGN.md §9.1`:
   - 4 variantes: `default` / `required` (asterisco accent-600) / `optional` (texto neutro) / `error` (accent-600)
   - Associado via `htmlFor`

7. **Badge** implementado conforme `DESIGN.md §9.1`:
   - Semânticas: `default` / `success` / `warning` / `error` / `info`
   - Estilos: `filled` / `outline` / `soft`

8. **Avatar** implementado conforme `DESIGN.md §9.1`:
   - Modos: `image` (com fallback) / `initials` (gerado automaticamente)
   - 6 tamanhos: `xs` / `sm` / `md` / `lg` / `xl` / `2xl`

9. **Icon** implementado como wrapper de Lucide React:
   - Props: `name` (string do Lucide) / `size` (px) / `className`
   - Tree-shakeable — não importa toda a lib

10. **Spinner** implementado conforme `DESIGN.md §9.1`:
    - 3 tamanhos: `sm` / `md` / `lg`
    - 2 cores: `primary` (primary-600) / `neutral` (neutral-400)
    - Animação `animate-spin`

11. **Barrel export** — `src/index.ts` exporta todos os 7 átomos nominalmente e como default de namespace:
    ```ts
    export { Button } from './atoms/Button'
    export { Input } from './atoms/Input'
    // ... etc
    ```

12. **Testes básicos** — cada átomo tem `.test.tsx` com no mínimo:
    - Renderiza sem erro (smoke test)
    - Variante/tamanho padrão renderiza corretamente
    - `aria-label` ou `role` acessível presente
    - Estado `disabled` funciona

13. **README.md** com:
    - Instruções de instalação (`package.json` dependency)
    - Configuração do Tailwind preset (`tailwind.config.js` do módulo consumidor)
    - Exemplo de uso de Button, Input e Avatar

## Notas técnicas

### Configuração do repo
- **PRÉ-CONDIÇÃO:** `@devops` cria o repo `ecossistema-ui` no GitHub ANTES de @dev inicializar (`gh repo create ewertonfm00/ecossistema-ui --public`)
- Criar como repo GitHub público separado: `ecossistema-ui`
- **NÃO** é um pacote npm — é instalado como git dependency
- TypeScript strict mode habilitado
- Path aliases: `@/` → `src/`

### Consumo nos módulos
```js
// tailwind.config.js de cada módulo
const aiox = require('@aiox/ui/tailwind-preset')
module.exports = {
  presets: [aiox],
  content: [
    './src/**/*.{ts,tsx}',
    './node_modules/@aiox/ui/src/**/*.{ts,tsx}', // incluir para purge funcionar
  ],
}
```

### Estrutura de componente (padrão a seguir)
```ts
// Button/Button.tsx
import { cva, type VariantProps } from 'class-variance-authority' // ou clsx
// ...
export interface ButtonProps
  extends React.ButtonHTMLAttributes<HTMLButtonElement>,
    VariantProps<typeof buttonVariants> {
  loading?: boolean
  leftIcon?: React.ReactNode
  rightIcon?: React.ReactNode
}
```

### Dependências recomendadas (a confirmar pelo @dev)
- `class-variance-authority` (cva) — para variantes de componente
- `clsx` + `tailwind-merge` — para merge de classes Tailwind
- `vitest` + `@testing-library/react` — para testes (alinhado ao ecossistema)

## Valor de negócio

Sem este repo, **nenhum módulo pode iniciar o desenvolvimento de UI**. É o desbloqueador de M1, M2 e M6. Cada hora investida aqui economiza semanas de inconsistência retroativa multiplicada por 5 módulos.
