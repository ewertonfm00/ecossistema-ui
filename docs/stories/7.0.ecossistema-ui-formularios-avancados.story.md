# Story 7.0 — Épico 09 — Formulários Avançados (FileUpload, MultiSelect, RangeSlider)

## Status: Draft

## Story

**As a** desenvolvedor construindo formulários dos módulos do ecossistema
**I want** componentes avançados de formulário (FileUpload com drag-and-drop, MultiSelect com tags, RangeSlider de intervalo)
**So that** posso compor fluxos de cadastro de pacientes, filtros multi-valor e seleção de faixas numéricas com UX profissional e sem dependências externas de UI

## Background

Os módulos do ecossistema precisam de padrões de formulário além dos átomos já entregues: upload de fotos antes/depois de procedimentos (FileUpload), seleção múltipla de categorias ou profissionais (MultiSelect), e configuração de faixas de preço ou horário (RangeSlider). Todos implementados sem biblioteca de terceiros além das já presentes (sem react-dropzone, sem react-select).

## Acceptance Criteria

### AC1 — FileUpload

- [ ] Aceita: `onFiles: (files: File[]) => void`, `accept?: string` (MIME type string, e.g. `"image/*,application/pdf"`), `multiple?: boolean` (padrão true), `maxSize?: number` (bytes, sem padrão — unlimited se omitido), `className?`
- [ ] Área de drop com borda dashed que muda para `primary-500` ao arrastar arquivo sobre ela
- [ ] Clique na área abre `<input type="file" hidden>` via `ref`
- [ ] Ao soltar arquivos, valida:
  - Tipo MIME contra `accept` (se fornecido) — arquivo rejeitado mostra mensagem "Tipo não permitido"
  - Tamanho contra `maxSize` (se fornecido) — arquivo rejeitado mostra "Arquivo muito grande (máx {n}MB)"
- [ ] Arquivos válidos são passados para `onFiles` e exibidos como lista de preview:
  - Para `image/*`: preview de thumbnail via `FileReader.readAsDataURL`
  - Para outros tipos: ícone genérico + nome do arquivo
- [ ] Botão `×` ao lado de cada item do preview remove o arquivo da lista
- [ ] Estado vazio: texto "Arraste arquivos aqui ou clique para selecionar"
- [ ] `role="region"` com `aria-label="Área de upload de arquivo"`

### AC2 — MultiSelect

- [ ] Aceita: `options: MultiSelectOption[]`, `value: string[]` (controlado), `onChange: (value: string[]) => void`, `placeholder?: string`, `searchable?: boolean` (padrão true), `disabled?: boolean`, `className?`
- [ ] Interface `MultiSelectOption = { value: string; label: string }`
- [ ] Renderiza valores selecionados como tags dentro do input (estilo chips)
- [ ] Cada tag tem botão `×` que remove o item individualmente via `onChange`
- [ ] Click no container abre dropdown com lista de opções
- [ ] Quando `searchable=true`: input de busca dentro do dropdown filtra opções por label (case-insensitive)
- [ ] Opção já selecionada aparece com checkmark e cor `primary-100` no dropdown
- [ ] Clique em opção não selecionada → adiciona ao array via `onChange`
- [ ] Clique em opção selecionada → remove do array via `onChange`
- [ ] Dropdown fecha ao clicar fora (useEffect + mousedown listener)
- [ ] Keyboard: `Escape` fecha dropdown, `Backspace` com input vazio remove última tag
- [ ] `aria-expanded`, `aria-haspopup="listbox"`, `role="option"`, `aria-selected`

### AC3 — RangeSlider

- [ ] Aceita: `value: [number, number]` (controlado), `defaultValue?: [number, number]`, `min: number`, `max: number`, `step?: number` (padrão 1), `onChange: (value: [number, number]) => void`, `formatValue?: (v: number) => string`, `label?: string`, `className?`
- [ ] Dois handles independentes: um para `min` e um para `max` do intervalo
- [ ] Implementação com dois `<input type="range">` sobrepostos (técnica de z-index):
  - Track base cinza atrás
  - Faixa ativa (entre os dois handles) em `primary-600`
  - Handle mínimo e máximo com estilos idênticos
- [ ] Valor mínimo nunca ultrapassa máximo; valor máximo nunca fica abaixo do mínimo
- [ ] Exibe valores formatados acima de cada handle (tooltip) via `formatValue` ou valor raw
- [ ] Uncontrolled: quando `value` não fornecido, usa `defaultValue` + estado interno
- [ ] `aria-label` em cada input: `"Valor mínimo"` / `"Valor máximo"`, com `aria-valuemin`, `aria-valuemax`, `aria-valuenow`

## Technical Notes

### FileUpload — Implementação de Drag-and-Drop

Usar eventos nativos HTML5 em uma `<div>`:
```tsx
onDragOver={(e) => { e.preventDefault(); setIsDragging(true) }}
onDragLeave={() => setIsDragging(false)}
onDrop={(e) => { e.preventDefault(); setIsDragging(false); handleFiles(e.dataTransfer.files) }}
```
`<input type="file" ref={inputRef} hidden accept={accept} multiple={multiple} onChange={...} />`

Preview de imagem com FileReader:
```ts
const reader = new FileReader()
reader.onload = (e) => setPreviewUrl(e.target?.result as string)
reader.readAsDataURL(file)
```

Validação:
```ts
function validateFile(file: File): string | null {
  if (accept && !accept.split(',').some(t => file.type.match(t.trim()))) return 'Tipo não permitido'
  if (maxSize && file.size > maxSize) return `Arquivo muito grande (máx ${Math.round(maxSize / 1024 / 1024)}MB)`
  return null
}
```

### MultiSelect — Técnica de Dropdown

- Estado: `isOpen: boolean`, `search: string`
- Ref no container + `useEffect` com `mousedown` para fechar ao clicar fora
- Track container via `containerRef.current.contains(event.target)`
- Filtro: `options.filter(o => o.label.toLowerCase().includes(search.toLowerCase()))`

### RangeSlider — Técnica de Dois Inputs Sobrepostos

```tsx
// Container com position: relative
// Ambos os inputs: position: absolute, width: 100%, height: 100%
// Input mínimo: z-index mais alto quando value[0] próximo ao máximo
// Track ativa calculada como % do min/max sobre o total (left%, right%)
```

Track ativa (estilo inline):
```tsx
const leftPct = ((effectiveMin - min) / (max - min)) * 100
const rightPct = 100 - ((effectiveMax - min) / (max - min)) * 100
// Div absoluta com left={leftPct}% right={rightPct}% bg-primary-600
```

## Dev Notes

- FileUpload: `FileReader` está disponível em JSDOM — testes podem usar arquivos mock via `new File(['...'], 'test.jpg', { type: 'image/jpeg' })`
- MultiSelect: não usar `select` nativo HTML — renderizar dropdown custom para suportar busca e chips
- RangeSlider: o CSS de sobreposição dos dois inputs requer que ambos tenham `pointer-events: none` com exceção do thumb — usar pseudo-selectors `input[type="range"]::-webkit-slider-thumb { pointer-events: all }`. Em Tailwind, isso exige classes arbitrárias ou CSS no `globals.css`. Usar `[&::-webkit-slider-thumb]:pointer-events-auto` e similares.
- Testes de drag-and-drop: `fireEvent.dragOver`, `fireEvent.drop` com `{ dataTransfer: { files: [file] } }` funcionam em @testing-library/react v14+
