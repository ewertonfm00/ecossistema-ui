## Notas tГ©cnicas

### Por que tsup e nГЈo tsc diretamente?

`tsc` gera apenas CJS. `tsup` usa esbuild internamente, gera ESM + CJS em paralelo, com `dts: true` delegando para `tsc` apenas os typings. Resultado: build 10-20x mais rГЎpido que `tsc` puro + ESM nativo.

### VerificaГ§ГЈo manual pГіs-build

```bash
npm run build
node -e "const { Button } = require('./dist/index.js'); console.log(typeof Button)"
# Output esperado: function
```

### `exports` map e compatibilidade

O campo `exports` tem prioridade sobre `main`/`module` em Node 16+ e bundlers modernos (Vite, webpack 5, esbuild). O fallback `main`/`module` garante compatibilidade com tooling antigo.
