# Story 6.0 — Épico 08 — Charts (BarChart, LineChart, PieChart, AreaChart)

## Status: Ready for Review

## Story

**As a** desenvolvedor integrando o design system nos módulos do ecossistema (Omega Laser, Estética IA, ML Laboratory)
**I want** componentes de gráfico prontos (BarChart, LineChart, PieChart/Donut, AreaChart) com visual consistente ao design system
**So that** posso exibir métricas de faturamento, ocupação e performance sem configurar recharts manualmente em cada módulo

## Background

Os dashboards do ecossistema precisam exibir dados de séries temporais (receita mensal, procedimentos por dia) e composições (distribuição de procedimentos, faturamento por categoria). O projeto usa recharts como única dependência aprovada para charts — os componentes deste épico são wrappers finos que injetam tokens do design system (paleta, tipografia, grid) e expõem API simplificada.

### Pattern de Render

```tsx
export function BarChart({ data, xKey, dataKeys, height = 300, stacked = false,
  showGrid = true, showLegend = true, showTooltip = true,
  colors, className }: BarChartProps) {
  const palette = colors ?? CHART_PALETTE
  return (
    <div role="img" aria-label="Gráfico de barras" className={cn('w-full', className)}>
      <ResponsiveContainer width="100%" height={height}>
        <RechartsBarChart data={data}>
          {showGrid && <CartesianGrid strokeDasharray="3 3" stroke="#e5e7eb" />}
          <XAxis dataKey={xKey} tick={{ fontSize: 12, fill: '#6b7280' }} />
          <YAxis tick={{ fontSize: 12, fill: '#6b7280' }} />
          {showTooltip && <Tooltip />}
          {showLegend && <Legend />}
          {dataKeys.map((key, i) => (
            <Bar key={key} dataKey={key} fill={palette[i % palette.length]}
              stackId={stacked ? 'stack' : undefined} radius={[4, 4, 0, 0]} />
          ))}
        </RechartsBarChart>
      </ResponsiveContainer>
    </div>
  )
}
```

## Dev Notes

- recharts v2.x inclui tipagens TypeScript próprias (sem `@types/recharts`)
- Em JSDOM (ambiente de teste), `getBoundingClientRect` retorna 0 — recharts pode renderizar SVG vazio; os testes devem verificar presença do elemento SVG, não dimensões
- `ResponsiveContainer` precisa de width/height definidos no container pai em testes; solução: `{ width: 500, height: 300 }` via `vi.spyOn` em `Element.prototype.getBoundingClientRect` OU simplificar e verificar apenas que o componente renderiza sem throw
- Connector de cores: não usar variáveis CSS do Tailwind diretamente (recharts usa SVG, não CSS classes) — usar hex hardcoded no `CHART_PALETTE`
