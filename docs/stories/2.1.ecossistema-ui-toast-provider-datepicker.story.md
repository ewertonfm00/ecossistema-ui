## Story

**Como** desenvolvedor de um módulo consumidor do ecossistema,
**quero** um sistema de notificações global via `ToastProvider` + `useToast` e um componente `DatePicker` com calendário dropdown,
**para que** eu possa notificar o usuário de ações assíncronas sem gerenciar estado de toast localmente, e coletar datas em formulários sem depender de `<input type="date">` nativo com inconsistências entre browsers.

---

## Contexto

O `Toast` átomo existente (Story 0.2) é um componente de notificação **inline** — exibe alertas dentro do fluxo da página. O `ToastProvider` desta story é um sistema diferente: um **singleton global** com portal React que gerencia uma fila de notificações posicionadas no canto da tela.

O `DatePicker` resolve o `<input type="date">` nativo que tem aparência e comportamento inconsistentes entre Chrome, Firefox e Safari. O componente expõe um input texto com máscara DD/MM/YYYY e um calendário dropdown completo.

**Decisão arquitetural:** `ToastProvider` usa Context API do React (sem Zustand — os módulos são responsáveis por seu estado, o `@aiox/ui` não deve forçar uma solução de estado global). `DatePicker` implementa o calendário com JavaScript puro (sem `date-fns` ou `dayjs` — o projeto não tem dependência de lib de data).

### 2. `ToastProvider` — sistema de toast global

#### 2a. Context e hook `useToast`

```ts
interface Toast {
  id: string
  message: string
  variant: 'success' | 'error' | 'warning' | 'info'
  duration?: number   // ms até auto-dismiss; padrão: 4000; 0 = persistente
  title?: string      // linha de título opcional acima da mensagem
}

interface ToastContextValue {
  addToast: (toast: Omit<Toast, 'id'>) => void
  removeToast: (id: string) => void
}

// Hook de consumo
export function useToast(): ToastContextValue
```

**Geração de id:** `crypto.randomUUID()` (disponível em browsers modernos e Node 18+) ou fallback `Date.now().toString(36) + Math.random().toString(36).slice(2)`.

#### 2b. `ToastProvider` component

```ts
interface ToastProviderProps {
  children: React.ReactNode
  position?: 'top-right' | 'top-left' | 'bottom-right' | 'bottom-left'  // padrão: 'top-right'
  maxToasts?: number  // máximo de toasts visíveis simultaneamente; padrão: 5
}
```

**Comportamento:**
- Renderiza `children` normalmente + portal com a fila de toasts no `document.body`
- `addToast` adiciona à fila; se `duration > 0`, agenda `removeToast` via `setTimeout`
- `removeToast` remove da fila pelo `id`
- Se fila exceder `maxToasts`, o mais antigo é removido automaticamente
- Ao desmontar um toast (remover da fila), executar `clearTimeout` pendente

**Posicionamento (classe CSS por `position`):**
| Valor | Classes |
|-------|---------|
| `top-right` | `fixed top-4 right-4` |
| `top-left` | `fixed top-4 left-4` |
| `bottom-right` | `fixed bottom-4 right-4` |
| `bottom-left` | `fixed bottom-4 left-4` |

**Container de toasts:**
```tsx
<div className={cn('z-[9999] flex flex-col gap-2 w-80', positionClass)} aria-live="polite" aria-atomic="false">
  {toasts.map(toast => <ToastItem key={toast.id} toast={toast} onClose={removeToast} />)}
</div>
```

#### 2c. `ToastItem` — item individual (componente interno)

**Variantes visuais:**
| `variant` | Ícone (`lucide-react`) | Borda esquerda |
|-----------|----------------------|----------------|
| `success` | `CheckCircle2` | `border-l-4 border-emerald-500` |
| `error` | `XCircle` | `border-l-4 border-red-500` |
| `warning` | `AlertTriangle` | `border-l-4 border-amber-500` |
| `info` | `Info` | `border-l-4 border-blue-500` |

**Estrutura:**
```tsx
<div
  role={variant === 'error' ? 'alert' : 'status'}
  aria-live={variant === 'error' ? 'assertive' : 'polite'}
  className="flex items-start gap-3 bg-white shadow-lg rounded-lg p-4 border border-neutral-200 ..."
>
  <Icon name={iconName} className={iconColorClass} size={20} aria-hidden="true" />
  <div className="flex-1 min-w-0">
    {title && <p className="text-sm font-semibold text-neutral-900">{title}</p>}
    <p className="text-sm text-neutral-600">{message}</p>
  </div>
  <button onClick={() => onClose(id)} aria-label="Fechar notificação" className="...">
    <Icon name="X" size={16} />
  </button>
</div>
```

**Acessibilidade (obrigatório):**
- `role="alert"` + `aria-live="assertive"` para variante `error`
- `role="status"` + `aria-live="polite"` para `success`, `warning`, `info`
- Container pai com `aria-live="polite"` + `aria-atomic="false"` (anuncia cada toast individualmente)
- Botão fechar com `aria-label="Fechar notificação"`

**Stories (Storybook):**
- `Default` — `ToastProvider` envolvendo botões que disparam cada variante
- `BottomRight` — `position="bottom-right"`
- `Persistent` — `duration={0}` (toast não fecha automaticamente)
- `MultipleToasts` — dispara 5 toasts rapidamente para mostrar a fila

**Testes (mínimo 2):**

### 3. `DatePicker` — input de data com calendário dropdown

**Interface:**
```ts
interface DatePickerProps {
  value?: Date | null
  onChange?: (date: Date | null) => void
  placeholder?: string              // padrão: 'DD/MM/AAAA'
  disabled?: boolean                // padrão: false
  minDate?: Date
  maxDate?: Date
  id?: string
  'aria-label'?: string
  'aria-describedby'?: string
}
```

#### 3a. Input com máscara

- Campo `<input type="text">` com placeholder `DD/MM/AAAA`
- Ícone de calendário (`CalendarDays` do lucide-react) no lado direito, clicável
- Máscara aplicada via `onKeyDown`/`onChange`: formata automaticamente para `DD/MM/AAAA` conforme o usuário digita
- Ao digitar data válida completa (10 chars), chama `onChange` com o `Date` correspondente
- Campo de texto reflete o `value` formatado: `value={formatDate(value)}` onde `formatDate` retorna `'DD/MM/AAAA'` ou `''`

**Validação do input de texto:**
- Aceitar apenas dígitos e `/` automáticos
- Data inválida (ex: `32/13/2026`) → não chama `onChange`, exibe borda vermelha
- Data fora de `minDate`/`maxDate` → não seleciona, exibe borda vermelha

#### 3b. Calendário dropdown

**Abertura:** clique no ícone de calendário ou no input abre o dropdown.
**Fechamento:** clique fora (blur do container), Escape, ou seleção de dia.

**Estrutura do calendário:**
```tsx
<div role="dialog" aria-label="Calendário" aria-modal="false" className="absolute z-50 mt-1 bg-white rounded-lg shadow-xl border border-neutral-200 p-3">
  {/* Header de navegação */}
  <div className="flex items-center justify-between mb-3">
    <button aria-label="Mês anterior" onClick={prevMonth}><Icon name="ChevronLeft" /></button>
    <span className="text-sm font-semibold">
      {monthName} {year}  {/* Ex: "Junho 2026" */}
    </span>
    <button aria-label="Próximo mês" onClick={nextMonth}><Icon name="ChevronRight" /></button>
  </div>
  {/* Cabeçalho dos dias da semana */}
  <div role="row" className="grid grid-cols-7 gap-1 mb-1">
    {['D','S','T','Q','Q','S','S'].map(d => (
      <span role="columnheader" aria-label={fullDayName} key={d} className="text-xs text-neutral-400 text-center">{d}</span>
    ))}
  </div>
  {/* Grid de dias */}
  <div role="grid" aria-label={`${monthName} ${year}`} className="grid grid-cols-7 gap-1">
    {days.map(day => (
      <button
        key={day.dateStr}
        role="gridcell"
        aria-selected={isSelected(day.date)}
        aria-label={day.ariaLabel}  // Ex: "15 de junho de 2026"
        aria-disabled={isDisabled(day.date)}
        onClick={() => !isDisabled(day.date) && selectDay(day.date)}
        className={cn(
          'h-8 w-8 rounded text-sm text-center transition-colors',
          isSelected(day.date) && 'bg-primary-600 text-white',
          isToday(day.date) && !isSelected(day.date) && 'border border-primary-600 text-primary-600',
          isDisabled(day.date) && 'text-neutral-300 cursor-not-allowed',
          !day.currentMonth && 'text-neutral-300',
        )}
      >
        {day.date.getDate()}
      </button>
    ))}
  </div>
</div>
```

**Navegação por teclado (obrigatório):**
- `ArrowLeft`/`ArrowRight`: move o foco entre dias
- `ArrowUp`/`ArrowDown`: move o foco semana para cima/baixo
- `Enter`/`Space`: seleciona o dia focado
- `Escape`: fecha o calendário sem selecionar
- `Home`/`End`: vai para primeiro/último dia da semana
- `PageUp`/`PageDown`: navega para mês anterior/próximo

## Notas técnicas

### ToastProvider — Context API sem Zustand

Usar Context API do React para o singleton de toasts:
```tsx
const ToastContext = React.createContext<ToastContextValue | null>(null)

export function useToast() {
  const ctx = React.useContext(ToastContext)
  if (!ctx) throw new Error('useToast deve ser usado dentro de ToastProvider')
  return ctx
}
```

Isso garante que os módulos consumidores (Estética IA, ML Lab) apenas envolvem o app com `<ToastProvider>` e chamam `useToast()` onde quiserem.

### ToastItem — componente interno, não exportado

`ToastItem` é um detalhe de implementação do `ToastProvider`. Não deve ser exportado no `src/index.ts`. Apenas `ToastProvider` e `useToast` são a API pública.

### DatePicker — comportamento de foco no calendário

O container do DatePicker deve ter `position: relative`. O dropdown do calendário tem `position: absolute` relativo ao container. Para detectar clique fora, usar `onBlur` no container com `e.relatedTarget` — se `relatedTarget` não está dentro do container, fechar o calendário.

```ts
function handleContainerBlur(e: React.FocusEvent) {
  if (!e.currentTarget.contains(e.relatedTarget as Node)) {
    setOpen(false)
  }
}
```

### Formato DD/MM/AAAA — parsing

```ts
function parseInput(str: string): Date | null {
  const match = str.match(/^(\d{2})\/(\d{2})\/(\d{4})$/)
  if (!match) return null
  const [, d, m, y] = match
  const date = new Date(+y, +m - 1, +d)
  // Verificar se a data é válida (ex: 31/02 criaria 03/03)
  if (date.getDate() !== +d || date.getMonth() !== +m - 1) return null
  return date
}
```

## Valor de negócio

`ToastProvider` é o mecanismo de feedback de ações assíncronas (salvar, deletar, erro de API) em todos os módulos. Sem ele, cada módulo implementa alertas próprios divergentes. `DatePicker` substitui o `<input type="date">` nativo que quebra em Safari e tem aparência inconsistente — crítico para agendamentos no módulo Estética IA.
